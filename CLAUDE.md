# go-trader Project Context

## Environment
- Go 1.26.2 (`brew install go@1.26` on macOS); use full binary path if not on PATH (e.g. `/opt/homebrew/bin/go`).
- Python venv at `.venv/bin/python3` (executor.go runtime); deps via `uv` (`pyproject.toml` / `uv.lock`).
- Git worktrees: `.venv` not copied — use `<main-repo>/.venv/bin/python3`.

## Setup
- `uv sync` — install Python deps
- Copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys
- New server: `install https://github.com/richkuo/go-trader and init`

## Repo Structure
- `scheduler/` — Go scheduler (single `package main`). Key files:

  - `server.go` — `/status`, `/health`, `/history`; `DefaultStatusPort=8099`, auto-fallback up to 5 ports; precedence `--status-port` > `cfg.StatusPort` > default.
  - `discord.go` — `discordgo.Session`; `SendMessage`/`SendDM`/`AskDM`; `FormatCategorySummary`. Trade DMs: `isTradeCloseDetails` (case-insensitive `"close"`) renders partial-close legs as `TRADE CLOSED` (#530); open DMs include extras line (Regime/ATR/TP1/TP2/SL, #561).

Perps `Leverage` is exchange/risk; `SizingLeverage` controls notional (`cash × sizing_leverage`, defaults to `Leverage`); `MarginPerTradeUSD` (`*float64`, #518) overrides: `notional = min(MarginPerTradeUSD, cash) × leverage`. `DefaultStopLossATRMult` top-level (`default_stop_loss_atr_mult`, default 1.0, #605) feeds omitted ATR stops. Five HL-perps stop fields (`StopLossPct`/`StopLossMarginPct`/`TrailingStopPct`/`TrailingStopATRMult`/`StopLossATRMult`, all `*float64`): mutually exclusive when positive; explicit `0` opts out; all-five-omitted (incl. shared-coin peers since #601) get top-level default; `default_stop_loss_atr_mult=0` opts the auto-default out fleet-wide (#562/#601/#605). `TrailingStopMinMovePct` debounce (default 0.5%); `MarginMode` (`isolated`|`cross`, default `isolated`); `RegimeConfig{Enabled,Period=14,ADXThreshold=20}`; `StrategyConfig.AllowedRegimes []string` entry gate (rejects unknown labels and `type=options`; warns when regime disabled globally, #541/#553/#558).

Tables: `app_state`, `strategies`, `positions`, `closed_positions`, `option_positions`, `closed_option_positions`, `trades`, `portfolio_risk`, `kill_switch_events`, `correlation_snapshot`, `pending_manual_actions` (#569). `app_state.last_summary_post` (#474) — wall-clock summary cadence per channel. `trades.position_id` / `positions.position_id` / `option_positions.position_id` (#471) group legs into round-trips via `idx_trades_strategy_position`. `trades.regime TEXT` (#542), `trades.entry_atr REAL` / `trades.stop_loss_trigger_px REAL` (#561), `trades.manual INTEGER` (#569), `pending_manual_actions.is_full_close INTEGER` (#569 — preserves operator intent past lot-size tolerance), `positions.tp_oids_json TEXT` (#615/issue #612 — full N-tier TP OID list; legacy `tp1_oid`/`tp2_oid` retained as compat shim via `firstTwoTPOIDs`/`parseTPOIDsJSON`). `InsertTrade` writes immediately. `CheckStatePresence` warns on missing DB (`GO_TRADER_ALLOW_MISSING_STATE=1` for first-run). All migrations idempotent via duplicate-column guard.
  - `risk.go` / `strategy_interval.go` — `CheckRisk` takes `*PlatformRiskAssist`; early-returns `true` for `sc.Type=="manual"` (#574). `perpsMarginDrawdownInputs` for leverage-aware perps DD. `effectiveStrategyIntervalSeconds` accelerates checks in DD warn band; `WarningSent` repeats while in band.

## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler && ...` to avoid cwd drift.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go` wizard + `generateConfig`, (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`/`get_real_expiry`/`get_real_strike`/`get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters via `importlib`; class detected by `endswith("ExchangeAdapter")` (one per file). Check scripts use only public adapter methods.
- **Close registry import:** never `import registry` directly (collides with open registry). Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts always emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of exit code.
- **State locking:** `mu sync.RWMutex` guards `state`. Per-strategy 6-phase loop: RLock(read) → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → RLock/no-lock/Lock(marks) → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)\|mu\.\(R\)\?Unlock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map: `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`, `manual` (#569). Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`. **Manual** has no script/interval — `LoadConfig` auto-fills `script`/`args[0]="hold"`/`interval_seconds`/SL@1×ATR + `tiered_tp_atr_live` (TP1@2× / TP2@3×); scheduler runs close-eval loop while position open and enqueues triggered closes back into `pending_manual_actions`. `hold` open strategy emits `signal=0`. `type=manual` may share a coin with another live `type=manual` or HL perps peer (#619/#620): owner-strategy guards on `runManualClose`/`applyManualAction`/main-loop manual case prevent cross-strategy mutation, full-close paths use `shouldCloseFullPosition` to fall back to a sized close when peers exist, and full-close cancels this strategy's `pos.TPOIDs`/`StopLossOID` via `extraCancelOIDs`. Peers must still agree on `leverage` and `margin_mode` (`hyperliquidPeerStrategyErrors` includes `type=manual`); at most one trailing-stop owner per coin. `CheckRisk` skipped for `type=manual` (#574).
- **Open/close split (#480/#483/#496):** `OpenStrategy` overrides entry name; `CloseStrategies` is ordered list, max `close_fraction` wins; empty falls back to open-as-close. Composition in `strategy_composition.{go,py}`. Opted-in scripts emit `open_action`, `close_fraction`, `close_strategy`, composed `signal`; close runs before open. Options not yet split. `Regime` propagated in all 5 result structs and written to `stratState.Regime` each cycle (#541). **Partial-close (#519/#521):** `close_fraction` honored everywhere; partial decrements `pos.Quantity`, preserves `InitialQuantity`, skips open-leg fall-through; legacy wrappers pass `0`. SL cancel/replace suppressed on partial. **Position-aware close evaluators (#496/#500):** check scripts forward `--position-{side,avg-cost,qty,initial-qty,entry-atr}` (Go: `PositionCtx`). Close functions opt in via kwargs or `**params`; `strip_unsupported_position_context` filters legacy; cache key includes merged params. Reference: `tp_at_pct`.
- **Live exec guard:** every dispatch in main.go uses `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** `runHyperliquidExecuteOrder` / `runOKXExecuteOrder` / `runRobinhoodExecuteOrder` / `runTopStepExecuteOrder` must check the same conditions as `ExecuteXxxSignal` BEFORE spawning — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** `AllowShorts` per-strategy. Strategies with short entries (`triple_ema_bidir`) must be in `bidirectionalPerpsStrategies` (init.go). `ExecutePerpsSignal` and `PerpsOrderSkipReason` must thread `sc.AllowShorts`. Live flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. Threads `pos.AvgCost` from Phase 1 snapshot.
- `dueStrategies` is value-copied from `cfg.Strategies` — mutations don't persist; update `cfg.Strategies` first.
- Notification features: add methods to `MultiNotifier`, don't access `backends`.
- HL sys.path conflict: SDK package `hyperliquid` clashes with `platforms/hyperliquid/` — add `platforms/hyperliquid/` directly to sys.path, then `from adapter import HyperliquidExchangeAdapter`.
- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); response uses parallel arrays.
- Fees: `CalculatePlatformSpotFee(platform,value)` (HL 0.035%, RH 0%, BinanceUS 0.1%); `CalculatePlatformFuturesFee(sc,contracts)`.
- Position ownership: `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles with owner only. **Shared-coin reconcile (#565/#617):** `reconcileHyperliquidAccountPositions` keeps the #258/#515 non-destructive default for ambiguous gaps but adds three unambiguous-close detectors:
  1. **Full external close** — on-chain qty ≈ 0 with non-zero virtual → close all peers (SL owner via `hl_sync_stop_loss` at trigger price; non-SL-owner peers via `recordPerpsExternalClose` with mark-based PnL credited to `s.Cash`, #584; zero-PnL fallback when no mark).
  2. **SL-owner partial close** — sole SL owner exists and on-chain residual matches signed sum of non-owner peers' virtual qty → close owner only (non-owners untouched).
  3. **TP partial fill (#617)** — `delta>0`, `hyperliquidSharedPartialCloseDrift` reports a single closeable side/qty, AND exactly one same-side strategy has a cleared on-chain TP tier per `hyperliquidHasClearedTPTier(sc,pos,closeQty)` (placed-tier OID now zero, so `applyHyperliquidProtectionSync` saw it filled externally) → book partial close via `recordPerpsExternalPartialCloseWithFillFee` and shrink virtual qty so next protection-sync sizes correctly. Multiple TP-tier candidates → leave gap visible.
  Reconciler close paths query real HL fee via `buildCachedHyperliquidReconcileFillResolver` (OID-first, coin+size fallback over 24h, built outside `mu.Lock`); failure → modeled fee (#588). `HLFillLookup.FilledQty` accumulated from fill `Sz` across all matched fills; when `useFillFee && FilledQty < virtualQty`, reconciler overrides `pos.Quantity` with actual fill size before PnL booking (#621/#622). **Reconciler scope (#576):** `hlReconcileAll`/`hlReconcileDue` built via `isHLLiveReconcilable` (HL + live + perps|manual) — `type=manual` positions closed externally are reconciled and cleared; `hlLiveAll`/`hlLiveDue` remain perps-only for kill-switch / trailing-stop / risk math.
- **State is SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). Trades persist via `RecordTrade` → `StateDB.InsertTrade`. `ClosedPositions` flushed by `SaveState`. Leaderboard on-demand (`BuildLeaderboardMessages`).
- **Trades schema (close legs):** `trades.is_close INTEGER` / `realized_pnl REAL` flag close legs; `trades.regime`, `trades.entry_atr`, `trades.stop_loss_trigger_px` stamped from in-scope `Position` at record time (`stampOpenTradeFromPosition` for open legs, #561). `stampOpenTradeFromPosition` is also called immediately when `StopLossTriggerPx` transitions from 0 to positive (both paper and live fixed ATR SL arm, #624) — backfills `stop_loss_trigger_px` for trades opened before the SL OID was set. `LifetimeTradeStatsAll(db)` drives `#T`/`W/L`. **`#T` counts positions opened (`is_close=0`), not round-trips (#608)** — `LifetimeTradeStats.PositionsOpened` (renamed from `RoundTrips`); opens contribute immediately. W/L still round-trip-grouped by `(strategy_id, position_id)`; wins use strict `realized_pnl > 0` (breakeven excluded). Partial-close legs collapse into one round-trip (#471); NULL/empty `position_id` rows each count as one. `FormatCategorySummary` renders zero when strategy absent — SQLite authoritative (#472). `SaveState` flush INSERT matches `InsertTrade` exactly (#561).
- **Live execute fills:** HL/OKX/RH/TopStep `--execute` scripts emit `fillFee` (and per-leg `fillFees`) plus exchange order IDs (`fillOID`/`exchange_order_id`); Go threads into `Trade.Fees` and `Trade.ExchangeOrderID` BEFORE `RecordTrade`. Python truthy filter (`if oid:`) — never `is not None` — so empty strings and numeric 0 don't land. TopStep precedence: prefer `orderId` even when numeric 0, fall back to `id` only when missing. Tests: `shared_scripts/test_check_execute_fees.py`. **HL real fee (#585/#587/#603):** adapter `lookup_fill_fee_by_oid` aggregates fee + closedPnl across partial fills sharing an OID, retries with backoff for indexer lag; wired into `check_hyperliquid.py` and `close_hyperliquid_position.py`; failure → modeled `CalculatePlatformSpotFee`. Pre-#587 rows with `exchange_fee=0` correctable via `go-trader backfill hl-fees` (#591/#602 widened lookup window). #600: `lookupHyperliquidFillByCoinSize` anchors on the newest matching record's OID group (was: sum every same-size match in window) so unrelated same-size closes don't conflate fees/PnL.
- **HL kill-switch fill accounting:** Shared-coin fills split by **virtual quantity at snapshot time** (not capital weight); snapshot under RLock BEFORE close mutates state. `hyperliquidKillSwitchFillShare` fails closed (returns `0,0`) when strategy isn't a peer or has no virtual qty.
- **Trade.Side on close legs:** records buy/sell semantics, not `"close"`. `closeTradeSide(positionSide)` (perps/spot — `"short"`→`"buy"`, else `"sell"`); `optionCloseTradeSide(action)` (options — `"sell"`→`"buy"`, else `"sell"`). Applies in `forceCloseAllPositions`, `recordPerpsStopLossClose`, `executeOptionClose`, `CheckThetaHarvest`. Default-to-`"sell"` preserves legacy.
- **TradingView export:** when user says "export data to TradingView", first ask which strategy IDs (or all). Use `./go-trader export tradingview --strategy <id> --output <file>` or `--all --output <file>`. Sourced from SQLite trades; may require `tradingview_export.symbol_overrides` for unmapped platforms/symbols.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting operator-facing or test-asserted output — Go map iteration is randomized.
- `cfg.Discord.Channels` / `Telegram.Channels` / `DMChannels` are `map[string]string`; keys: `spot`, `options`, `<platform>`, `<platform>-paper`. `cfg.Discord.OwnerID` from `DISCORD_OWNER_ID` env (priority over config).
- `cfg.Discord.TradeAlertChannels` / `cfg.Telegram.TradeAlertChannels` (`map[string]string`, optional, #572) — override channels for trade-fill alerts; same key scheme; falls back to `Channels`. Summaries always use `Channels`. SIGHUP-reloadable. A `stratType` key (e.g. `"perps"`) reroutes that type across **all** platforms — use a platform key for per-platform.
- `cfg.SummaryFrequency map[string]string`: Go duration / alias (`hourly`, `daily`, `every`/`per_check`/`always`) / empty (legacy). `ShouldPostSummary(freq,continuous,hasTrades,lastPost,now)` — wall-clock throttling (#474); `hasTrades=true` always forces a post. `lastSummaryPost` tracked single-writer in main loop, mirrored into `state.LastSummaryPost`, persisted in `app_state.last_summary_post`. Use `cloneTimeMap` between local map and `AppState`.
- `cfg.Correlation` (`Enabled=false`, `MaxConcentrationPct=60`, `MaxSameDirectionPct=75`); `cfg.AutoUpdate`: `off|daily|heartbeat`.
- **Regime detection (#539–#558):** `shared_tools/regime.py` 3-state labels (`trending_up`/`trending_down`/`ranging`) via ADX+DI; canonical set in `validRegimeLabels` (`config.go`). All 5 standard check scripts inject label; `check_options.py` emits `"regime": null` (deferred). `"regime"` is in `POSITION_CONTEXT_PARAM_KEYS` so `strip_unsupported_position_context` drops it for legacy strategies. Go appends `--regime-{enabled,period,adx-threshold}` via `appendRegimeArgs` (#558); absent → `regime=""`. Entry gate `regimeBlocksOpen(allowed,current,posQty)` — close legs always pass. `AllowedRegimes` SIGHUP-compatible; `cfg.Regime` block requires restart.
- **Strategy registries (#511):** open implementations live once under `shared_strategies/open/`, source of truth `open/registry.py`; spot/futures shims build fresh `STRATEGY_REGISTRY` via `build_registry()`. Close evaluators: `shared_strategies/close/registry.py` with `evaluate(position,market,params)`. New open strategy: (1) register in `open/registry.py` + `PLATFORM_ORDER`, (2) `knownShortNames` + defaults in `init.go`, (3) `DEFAULT_PARAM_RANGES` in `backtest/optimizer.py`. Use `variants={"futures":{...}}` not duplication. Before refactoring registry: snapshot `--list-json` output and `diff` after — Go `discoverStrategies` depends on byte-identical output.
- `apply_strategy(name,df,params)` — config params merged UNDER runtime params (runtime priority).
- `check_strategy.py` uses manual `sys.argv` parsing — filter `--`-prefixed args before indexing. Other check scripts use argparse.
- `StrategyConfig.HTFFilter` — Go appends `--htf-filter`; not applied to options or `delta_neutral_funding`.
- `delta_neutral_funding` is perps-only — function in `spot/strategies.py` but registered only for futures.
- Perps vs futures at Position level: both `Multiplier > 0`; only perps `Leverage > 0`.
- Per-strategy DD: spot/options/futures peak-relative; perps `unrealized_loss / deployed_margin` when positions open. Margin uses exchange `Leverage`, not `SizingLeverage` (#497). `RecordTradeResult` only updates `DailyPnL` and `ConsecutiveLosses` — lifetime trade counts come from SQLite (#472).
- **Test pattern:** extract pure helpers from subprocess-spawning wrappers — Go CI lacks `.venv/bin/python3` so any test calling `RunPythonScript`/`RunHyperliquidExecute`/`RunHyperliquidClose` fails CI.
- Per-strategy CB pending close: `RiskState.PendingCircuitCloses map[string]*PendingCircuitClose` keyed by `PlatformPendingClose{Hyperliquid,OKX,Robinhood,TopStep,OKXSpot,RobinhoodOptions}`. Use `setPendingCircuitClose`/`clearPendingCircuitClose`/`getPendingCircuitClose`. ## Build & Deploy
