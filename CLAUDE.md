# go-trader Project Context

## Environment
- Go 1.26.2 (`brew install go@1.26` on macOS); use full binary path if not on PATH (e.g. `/opt/homebrew/bin/go`).
- Python venv at `.venv/bin/python3` (executor.go runtime); deps via `uv` (`pyproject.toml` / `uv.lock`).
- Git worktrees: `.venv` not copied — use `<main-repo>/.venv/bin/python3`.

## Setup
- `uv sync` — install Python deps
- Copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys
- New server: `install https://github.com/richkuo/go-trader and init`

## Repo Structure
- `scheduler/` — Go scheduler (single `package main`). Key files:
  - `executor.go` / `shutdown.go` — Python subprocess runner; `pythonSemaphore=4`, `scriptTimeout=30s`, SIGKILLs on timeout. Two SIGTERM-aware wrappers: `runPythonReadOnly` (cancelled immediately) vs `runPythonSideEffect` (registers on `sideEffectWG`; drain waits for in-flight on-chain mutations). New side-effecting wrapper → use `runPythonSideEffect`, never `runPython`. `shutdown.go` two-phase drain: signal goroutine `beginDrain` (atomic flag + cancel read-only ctx); main defers `runDrain` (waits `shutdownDrainCap=15s` then SIGKILL); state-save/notifier-flush/DB-close runs after via deferred LIFO. CLI/test paths skip `initShutdownContexts` → `context.Background()` defaults.
  - `server.go` — `/status`, `/health`, `/history`; `DefaultStatusPort=8099`, auto-fallback up to 5 ports; precedence `--status-port` > `cfg.StatusPort` > default. `/health` JSON includes `version` (used by `update.sh` post-restart verify); returns 503 `{"status":"draining"}` once `isDraining()`.
  - `discord.go` — `discordgo.Session`; `SendMessage`/`SendDM`/`AskDM`; `FormatCategorySummary` cols `Init|Value|PnL|PnL%|DD|Wallet%|Tf|Int|#T|W/L` + `Book Sharpe` footer. `collectPositions` appends extras `ATR | SL [×Nmult] | TP[i] | leverage`; Nmult reads stamped `*float64` (`StopLossATRMult`/tier `Multiplier`), nil drops the suffix — back-computing from rounded trigger px drifts. Filled TP tiers marked `✓` when `pos.TPOIDs[i]==0` AND `pos.Quantity < pos.InitialQuantity` (both required — OIDs zero pre-protection-sync). Partial closes render `TRADE CLOSED`. TP prices via shared `tieredTPATRPrices(sc, side, entryPx, entryATR)` in `hyperliquid_protection.go` (`telegram.go` shares it). `tradeAlertExtras` appends `| OID: <id>` on live fills (paper omits) + `(<n>x)` ATR-mult on SL/TPs (`%g`, same stamped source).
  - `init.go` — `go-trader init` wizard + `--json` mode; `generateConfig(InitOptions)` is pure. Holds `bidirectionalPerpsStrategies`, `knownShortNames`, `default{Spot,Perps,Futures}Strategies`.

Perps `Leverage` (exchange/risk) vs `SizingLeverage` (notional, `cash × sizing_leverage`); `MarginPerTradeUSD` overrides as `min(MarginPerTradeUSD, cash) × leverage`. Five HL stop fields (`StopLossPct`/`StopLossMarginPct`/`TrailingStopPct`/`TrailingStopATRMult`/`StopLossATRMult`, `*float64`, mutually exclusive when positive) — priority in HL perps stop-loss section. `DefaultStopLossATRMult` (default 1.0) feeds all-five-omitted; `0` opts out fleet-wide. `TrailingStopMinMovePct` debounce (0.5%); `MarginMode` (`isolated`|`cross`); `RegimeConfig{Enabled,Period=14,ADXThreshold=20}`; `StrategyConfig.AllowedRegimes []string` entry gate.
    **Direction enum (v14):** `StrategyConfig.Direction` (`"long"`|`"short"`|`"both"`) replaces legacy `AllowShorts` bool; legacy field retained for unmarshal but accessed only via `EffectiveDirection`/`PerpsAllowsLong`/`PerpsAllowsShort`. `migrateV14Direction` rewrites `false→"long"`, `true→"both"`. `direction="short"` opens shorts on `signal=-1`, closes on `signal=1`, never flips long; orphan-direction signals skipped in both `PerpsOrderSkipReason` and `ExecutePerpsSignalWithLeverage`. `manual-open` rejects `--side` mismatches; init wizard emits `Direction` (no auto "short").

New close evaluator that owns params → update `closeStrategyOwnedKeys`; `TestCloseStrategyOwnedKeysMirrorsPythonRegistry` enforces Python parity.
  - `hyperliquid_trailing_stop.go` — synthetic `%` and ATR-mult trailing: `computeTrailingStopUpdate` walks high-water mark, emits cancel/replace when trigger moves favorably by `TrailingStopMinMovePct`; `runHyperliquidTrailingStopUpdate` warns on cancel failure (HL 1000-trigger cap race).

All migrations idempotent via duplicate-column guard.

Resolver seeds `(coin, 0, virtual−onChain)` for partial-drop matching. `hlAttemptCloseFromTPFills` (`hyperliquid_balance.go`) called from `syncHyperliquidAccountPositions` on vanish: SL fills first (gated `useFillFee && lookup.OID==pos.StopLossOID && FilledQty>0` — coin+size hits excluded, else same-size TP fills starve TP attribution), else TP-OID fills per tier at VWAP. Off-chain SL fallback at `:527` enforces the same OID gate; non-match → `hl_sync_external` at zero PnL with routing-decision INFO log. Prevents mis-attributing TP-fired closes (incl. all-TPOIDs-zeroed) to the SL trigger after HL auto-cancels the resting SL.
  - `backfill_hl_fees.go` — `go-trader backfill hl-fees [--strategy <id>|--all] [--apply] [--reset-cash]`: rewrites historical `exchange_fee=0` rows, adjusts `realized_pnl`, replays `strategies.cash`; dry-run by default; refuses `--apply` when another go-trader is alive.

Sizing: `--size`/`--notional`/`--margin` (mutually exclusive). ## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler &&`.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go` wizard + `generateConfig`, (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`/`get_real_expiry`/`get_real_strike`/`get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters via `importlib`; class detected by `endswith("ExchangeAdapter")` (one/file). Check scripts use public methods only.
- **Close registry import:** never `import registry` directly (collides with open registry). Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of code.
- **State locking:** `mu sync.RWMutex` guards `state`. Per-strategy 6-phase loop: RLock(read) → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → RLock/no-lock/Lock(marks) → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)\|mu\.\(R\)\?Unlock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map: `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`, `manual`. Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`. **Manual** has no script/interval — `LoadConfig` auto-fills `script`/`args[0]="hold"` (emits `signal=0`)/`interval_seconds`/SL@1×ATR + `tiered_tp_atr_live` (TP1@2× / TP2@3×); scheduler runs close-eval while position open and enqueues triggered closes into `pending_manual_actions`. `CheckRisk` skipped. Manual may share a coin with another manual or HL perps peer — owner guards on `runManualClose`/`applyManualAction`/main-loop manual case prevent cross-strategy mutation, full-close uses `shouldCloseFullPosition` + `extraCancelOIDs`.
- **Open/close split:** `OpenStrategy` (StrategyRef) overrides entry name; `CloseStrategies` (`[]StrategyRef`) ordered, max `close_fraction` wins; empty falls back to open-as-close. Composition in `strategy_composition.{go,py}`. Per-close knobs (e.g. `tiered_tp_atr.tiers`) stay on each ref's `Params`. Go emits `--strategy-refs <json>` (`{"open":{name,params},"close":[{name,params}…]}`); Python `parse_strategy_refs_arg` + `evaluate_open_close(close_params_by_name)`. Duplicate close-ref names: last write wins. Opted-in scripts emit `open_action`/`close_fraction`/`close_strategy`/composed `signal`; close runs before open. Options not yet split. `Regime` propagated in all 5 result structs → `stratState.Regime` each cycle. **Partial-close:** `close_fraction` decrements `pos.Quantity`, preserves `InitialQuantity`, skips open-leg fall-through; legacy wrappers pass `0`; SL cancel/replace suppressed on partial. **Position-aware close evaluators:** scripts forward `--position-{side,avg-cost,qty,initial-qty,entry-atr}` (Go `PositionCtx`). Close functions opt in via kwargs or `**params`; `strip_unsupported_position_context` filters legacy; cache key includes merged params. Reference: `tp_at_pct`.
- **Live exec guard:** every dispatch in main.go uses `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** `runHyperliquidExecuteOrder` / `runOKXExecuteOrder` / `runRobinhoodExecuteOrder` / `runTopStepExecuteOrder` must check the same conditions as `ExecuteXxxSignal` BEFORE spawning — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** per-strategy `Direction` enum (see config bullet). Short-entry strategies (`triple_ema_bidir`, `bear_pullback_st`, `vwap_rejection_st`, `donchian_breakout`, `chart_pattern`, `liquidity_sweeps`) must be in `bidirectionalPerpsStrategies` (init.go). `ExecutePerpsSignalWithLeverage` + `PerpsOrderSkipReason` thread the resolved direction. Live flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. Threads `pos.AvgCost` from Phase 1 snapshot. `ValidatePerpsDirectionConfig` flags both gap classes.
- `dueStrategies` is value-copied from `cfg.Strategies` — mutations don't persist; update `cfg.Strategies` first.
- Notification features: add methods to `MultiNotifier`; don't access `backends`.
- HL sys.path: SDK `hyperliquid` clashes with `platforms/hyperliquid/` — add `platforms/hyperliquid/` directly to sys.path, then `from adapter import HyperliquidExchangeAdapter`.
- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); parallel arrays.
- Fees: `CalculatePlatformSpotFee(platform,value)` (HL 0.035%, RH 0%, BinanceUS 0.1%); `CalculatePlatformFuturesFee(sc,contracts)`.
- Position ownership: `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles owner only. **Shared-coin reconcile:** `reconcileHyperliquidAccountPositions` is non-destructive for ambiguous gaps; three unambiguous-close detectors:
  1. **Full external close** — on-chain qty ≈ 0 with non-zero virtual → close all peers (SL owner via `hl_sync_stop_loss`; non-SL-owners via `recordPerpsExternalClose`, mark-based PnL to `s.Cash`).
  2. **SL-owner partial close** — sole SL owner; on-chain residual matches signed sum of non-owner peers → close owner only.
  3. **TP partial fill** — `delta>0`, `hyperliquidSharedPartialCloseDrift` reports one closeable side/qty AND exactly one same-side strategy has a cleared TP tier per `hyperliquidHasClearedTPTier` → book via `recordPerpsExternalPartialCloseWithFillFee` and shrink virtual qty. Multiple candidates → leave gap visible.
  Close paths use real HL fee via `buildCachedHyperliquidReconcileFillResolver` (OID-first, coin+size 24h fallback); failure → modeled fee. When `useFillFee && FilledQty < virtualQty`, override `pos.Quantity` with actual fill size before PnL booking. **Scope:** `hlReconcileAll`/`Due` via `isHLLiveReconcilable` (HL + live + perps|manual); `hlLiveAll`/`Due` perps-only (kill-switch / trailing / risk).
  **Owner DM on TP/SL fill:** all three detectors append `ProtectionFillAlert` to `pendingAlerts` while holding `mu`; deferred closure registered BEFORE `defer mu.Unlock()` flushes via `protection_fill_alert.go`'s `notifyProtectionFill` after unlock (Discord HTTP must not stall `/status`/per-strategy RLocks). TP tier from `hyperliquidClearedTPTier(sc, pos, closeQty)` (first cleared when multiple shrank). Toggle: `notify_tp_sl_fills` (`*bool`, nil→on); read via `Config.NotifyTPSLFillsEnabled()`.
  **Sole-owner TP attribution (non-shared path):** `reconcileHyperliquidPositionsForStrategy` → `tryBookSoleOwnerTPFill` books partial drift or final-tier flatten at config TP price (or `HLFillLookup.Px` when lookup hits) via `recordPerpsExternalPartialCloseWithFillFee` + `ProtectionFillAlert`. **Full-close requires ALL `TPOIDs[i]==0`** — any active tier defers to the legacy SL branch so a residual closed by SL/operator/CB after a prior partial TP doesn't mis-attribute. `applyHyperliquidProtectionSync` zeros `TPOIDs[i]` cycle N → attribution fires N+1.
- **State is SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). Trades persist via `RecordTrade` → `StateDB.InsertTrade`. `ClosedPositions` flushed by `SaveState`. Leaderboard on-demand (`BuildLeaderboardMessages`).
- **Trades schema:** `trades.is_close`/`realized_pnl` flag close legs; close-leg `regime`/`entry_atr`/`stop_loss_trigger_px` stamped from in-scope `Position` via `stampOpenTradeFromPosition`. **Open-trade snapshot:** `recordPositionOpen(s, sc, trade, pos)` writes one INSERT with full protection snapshot via `stampPositionProtectionSnapshot` + `copyPositionOpenSnapshotToTrade`. Deferred-open variants `Execute{Spot,Perps,Futures}*DeferredOpen` return `SignalExecutionResult{TradesExecuted, OpenTrade}`; main loop runs `runHyperliquidProtectionSync` between them and `recordPositionOpen` so SL/TP OIDs land same-cycle. `stampOpenTradeFromPosition` is fallback for late-armed protection. `LifetimeTradeStatsAll(db)` drives `#T`/`W/L`: `#T` counts opens (`is_close=0`); W/L round-trip-grouped by `(strategy_id, position_id)`, wins strict `realized_pnl > 0`; partial-close legs collapse into one round-trip; NULL `position_id` rows count individually. `SaveState` flush INSERT matches `InsertTrade` exactly.
- **Live execute fills:** HL/OKX/RH/TopStep `--execute` scripts emit `fillFee` (per-leg `fillFees`) + exchange order IDs (`fillOID`/`exchange_order_id`); Go threads into `Trade.Fees`/`Trade.ExchangeOrderID` BEFORE `RecordTrade`. Python truthy filter `if oid:` drops empty strings/0. TopStep prefers `orderId` (even 0), falls back to `id`. **HL real fee:** `lookup_fill_fee_by_oid` aggregates fee+closedPnl across partial fills sharing an OID with backoff; failure → modeled `CalculatePlatformSpotFee`. Historical `exchange_fee=0` rows correctable via `go-trader backfill hl-fees`. `lookupHyperliquidFillByCoinSize` anchors on the newest match's OID group so unrelated same-size closes don't conflate fees/PnL.
- **HL kill-switch fill accounting:** shared-coin fills split by **virtual qty at snapshot time** (not capital weight); snapshot under RLock BEFORE close mutates state. `hyperliquidKillSwitchFillShare` fails closed (`0,0`) when strategy isn't a peer or has no virtual qty.
- **Trade.Side on close legs:** records buy/sell semantics, not `"close"`. `closeTradeSide(positionSide)` (perps/spot — `"short"`→`"buy"`, else `"sell"`); `optionCloseTradeSide(action)` (options — `"sell"`→`"buy"`, else `"sell"`). Applies in `forceCloseAllPositions`/`recordPerpsStopLossClose`/`executeOptionClose`/`CheckThetaHarvest`.
- **TradingView export:** when user says "export data to TradingView", first ask which strategy IDs (or all). Use `./go-trader export tradingview --strategy <id> --output <file>` or `--all --output <file>`. Sourced from SQLite trades; may require `tradingview_export.symbol_overrides` for unmapped platforms/symbols.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting operator-facing or test-asserted output — Go map iteration is randomized.
- `cfg.Discord.Channels` / `Telegram.Channels` / `DMChannels` are `map[string]string`; keys: `spot`, `options`, `<platform>`, `<platform>-paper`. `cfg.Discord.OwnerID` from `DISCORD_OWNER_ID` env (priority over config).
- `cfg.Discord.TradeAlertChannels` / `cfg.Telegram.TradeAlertChannels` (`map[string]string`, optional) — override channels for trade-fill alerts; same key scheme; falls back to `Channels`. Summaries always use `Channels`. SIGHUP-reloadable. A `stratType` key (e.g. `"perps"`) reroutes that type across **all** platforms — use a platform key for per-platform.
- `cfg.SummaryFrequency map[string]string`: Go duration / alias (`hourly`, `daily`, `every`/`per_check`/`always`) / empty (legacy). `ShouldPostSummary(freq,continuous,hasTrades,lastPost,now)` wall-clock throttles; `hasTrades=true` always forces a post. `lastSummaryPost` is single-writer in main loop, mirrored into `state.LastSummaryPost`, persisted in `app_state.last_summary_post` (use `cloneTimeMap`).
- `cfg.Correlation` (`Enabled=false`, `MaxConcentrationPct=60`, `MaxSameDirectionPct=75`); `cfg.AutoUpdate` (`off|daily|heartbeat`).
- **Regime detection:** `shared_tools/regime.py` 3-state labels (`trending_up`/`trending_down`/`ranging`) via ADX+DI; canonical set in `validRegimeLabels`. All 5 standard check scripts inject label; `check_options.py` defers (`"regime": null`). `"regime"` in `POSITION_CONTEXT_PARAM_KEYS` so `strip_unsupported_position_context` drops it for legacy strategies. Entry gate `regimeBlocksOpen(allowed,current,posQty)` — close legs always pass. `AllowedRegimes` SIGHUP-compatible; `cfg.Regime` block requires restart.

## Build & Deploy
- **Update (operator + agents):** `bash scripts/update.sh --restart`. Atomic flow: `preflight` (hard-fail only on dirty build-input paths `scheduler/`/`shared_*`/`platforms/`/`backtest/`/`pyproject.toml`/`uv.lock`) → `pull --ff-only` → `uv sync` → build to `go-trader.new` → `probe` (`./go-trader.new probe`) → atomic swap with `.prev` retention → `restart`+`verify` (poll `systemctl is-active` + `/health` `version` match AND `MainPID` ≠ pre-restart PID). On timeout/mismatch `do_rollback` resets git to `pre_pull_sha`, re-syncs uv if HEAD advanced, restores `.prev`. `restart`+`verify` only with `--restart`/`RESTART=1`. Phases prefixed `[update] phase: <name>`. Env: `STATUS_PORT` (default from `scheduler/config.json`), `ACTIVE_TIMEOUT=30s`, `HEALTH_TIMEOUT=60s`. `applyUpgrade` calls same script without `--restart`. **Do not** rebuild Go in isolation — argv contract (`--strategy-refs`, `--probe-only`) needs both sides at same SHA. CLI: `./go-trader probe [--config ...]`; `./go-trader version` (empty on pre-#682 → verify falls back to version-match alone).
- **Graceful shutdown:** `systemctl restart` drains side-effecting subprocesses up to `shutdownDrainCap=15s` then SIGKILLs; state-save / notifier flush / DB close run after. Unit `TimeoutStopSec=20`. Journal: `[shutdown] draining` / `State saved` / `Complete`. `/health` 503 while draining. Service-file change requires `daemon-reload`.
- Build (one-off, no update): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (without `-ldflags` summary header shows `(dev)`). Always rebuild before smoke-testing.
- Restart: `systemctl restart go-trader` (service-file changes: `daemon-reload` first). Config-only: `kill -HUP $(pgrep go-trader)`. Python changes take effect next cycle.
- **Startup compatibility probe:** binary invokes each unique check script with `--probe-only` (see `probeCheckScripts`); non-zero exit → log + owner DM + `os.Exit(1)`. If startup fails post-update, ensure `git pull` updated `shared_scripts/` and rebuild — don't bypass.
