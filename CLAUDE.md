# go-trader Project Context

## Environment
- Go 1.26.2; full binary path if not on PATH (`/opt/homebrew/bin/go`).
- Python via `uv run --no-sync python` for dev/backtest/manual CLI; Go subprocess (scheduler) calls `.venv/bin/python3` directly — deterministic after `uv sync`, no systemd PATH config. Worktrees: `uv sync` once per checkout.
- **systemd:** `go-trader.service`, `systemd/go-trader@.service` use `ProtectSystem=strict`; no `PATH`/`UV_CACHE_DIR` injection. `/opt/go-trader/.env` (or per-instance) sets `DISCORD_BOT_TOKEN` + secrets.

## Setup
- `uv sync`; copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys.

## Decision Priorities
- **Pick the best technical solution.** Tests, code volume, time-to-implement are NOT design constraints — update/rewrite tests after the feature is done. These govern *quality*, not *scope*; don't override the branch+PR workflow, "verify issue claims against code", or destructive-action safety rules.

## Repo Structure

Co-located `OpenStrategy`+`CloseStrategies`; **new close evaluator → `closeStrategyOwnedKeys`**. Composite `sl_after` labels resolve via `regime_atr_window`; new surface → baseline + `closeStrategyOwnedKeys`.

## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler &&`.
- **New platform (8):** (1) `adapter.py`+`__init__.py`, (2) `check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix+validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go`+`generateConfig`, (8) `pyproject.toml`. New options: adapter `get_{vol_metrics,real_expiry,real_strike,premium_and_greeks}` + `check_options.py` + `CalculateOptionFee` + `OptionPlatforms`.
- Adapters via `importlib`, class `endswith("ExchangeAdapter")` (one/file); check scripts use public methods only.
- **Close registry import:** never `import registry` directly (collides w/ open) — use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`).
- Subprocess contract: scripts emit JSON to stdout even on error; exit 1 on error; Go parses regardless of code.
- **State locking:** `mu sync.RWMutex`. 6-phase loop: RLock → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → marks → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types `spot`,`options`,`perps`,`futures`,`manual`. Perps paper reuses `ExecuteSpotSignal`, live calls `RunHyperliquidExecute` first; futures `ExecuteFuturesSignal`. **Manual** auto-fills `script`/`args[0]="hold"`/interval/SL@1.5×ATR + `tiered_tp_atr_live`; `CheckRisk` skipped; shared-coin peers guarded by owner checks.
- **Open/close split:** `OpenStrategy` overrides entry; `CloseStrategies` ordered, max `close_fraction` wins; empty → open-as-close (`--strategy-refs <json>`); close before open; options not split. **Partial-close** decrements `pos.Quantity`, preserves `InitialQuantity`, suppresses SL cancel/replace. Position-aware via `--position-*`/`PositionCtx`; `def *_strategy` must name framework kwargs explicitly.
- **Live exec guard:** every main.go dispatch uses `liveExecFailed`; `runXxxExecuteOrder` returning `ok2=false` → skip state update.
- **Skip-reason guards:** `run{Hyperliquid,OKX,Robinhood,TopStep}ExecuteOrder` must repeat `ExecuteXxxSignal`'s conditions **before spawning** (else on-chain fill, no Trade record). Use `{Perps,Spot,Futures}OrderSkipReason`; capture `posSide` with `posQty` in Phase 1 RLock (`Position.Quantity` always positive).
- **Bidirectional perps:** short-entry strategies (`triple_ema_bidir`,`bear_pullback_st`,`vwap_rejection_st`,`donchian_breakout`,`chart_pattern`,`liquidity_sweeps`) must be in `bidirectionalPerpsStrategies` (init.go). Flip sizing `perpsLiveOrderSize`; catastrophic flip → close-only.
- `dueStrategies` value-copied from `cfg.Strategies` — mutations don't persist; update `cfg.Strategies` first.
- Notifications: add methods to `MultiNotifier` (not `backends`); guard new backends behind nil-checks. Channels keyed `spot`/`options`/`<platform>`/`<platform>-paper` (+`*.TradeAlertChannels` overrides); `stratType` reroutes across platforms; SIGHUP-reloadable.
- HL: add `platforms/hyperliquid/` to sys.path before `from adapter import HyperliquidExchangeAdapter` (SDK clash); funding rates `info.meta_and_asset_ctxs()` NOT `info.meta()`. Fees `CalculatePlatform{Spot,Futures}Fee`.
- **Position ownership:** `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles owner only; shared-coin reconcile non-destructive for ambiguous gaps. **SL attribution** `hlReconcileSLFillConfirmed` (exact OID+qty) else `hl_sync_external`.
- **State SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). `RecordTrade`→`StateDB.InsertTrade`; `ClosedPositions` flushed by `SaveState`; leaderboard `BuildLeaderboardMessages`.
- **Trades schema:** `is_close`/`realized_pnl` flag close legs; `DeferredOpen` runs protection-sync before `RecordTrade`. `LifetimeTradeStatsAll` `#T` counts opens, W/L grouped by `(strategy_id, position_id)`. HL real fee `lookup_fill_fee_by_oid` (failure → modeled). **`HLFillLookup.ClosedPnLGross` is gross — never wire into `Trade.RealizedPnL`.**
- **HL kill-switch fill accounting:** shared-coin fills split by virtual qty at snapshot time (RLock BEFORE close mutates); `hyperliquidKillSwitchFillShare` **fails closed** when not a peer. **Close-side:** `closeTradeSide` short→`buy` else `sell`; `optionCloseTradeSide` sell→`buy` else `sell`.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting operator-facing or test output — Go map iteration is randomized.
- **Regime detection** (`shared_tools/regime.py`): `adx` default (3 labels) / `composite` opt-in (7 labels); `regime.windows` bare int → `{classifier:"adx",period:N}` or `{classifier,period,thresholds?}`. `validateStrategyRegimeVocabulary` matches `allowed_regimes`/`regime_directional_policy`/`*_atr_regime` keys to each window's classifier. SIGHUP: window edits when flat; classifier swap/removal on a referenced window blocked while open. Multi-window primary: window `medium` or first sorted key.
- **Strategy registries:** new open strategy → (1) `open/registry.py`+`PLATFORM_ORDER`, (2) `knownShortNames`+defaults in `init.go`, (3) `DEFAULT_PARAM_RANGES` in `backtest/optimizer.py`. Use `variants={"futures":{...}}`. **Before refactoring registry: snapshot `--list-json` and diff after — Go `discoverStrategies` needs byte-identical output.**
- Strategy DSL: `apply_strategy` merges config params UNDER runtime (runtime wins); `HTFFilter` not on options/`delta_neutral_funding` (perps-only); perps & futures `Position` have `Multiplier>0`, only perps `Leverage>0`.
- **Per-strategy CB pending close:** `RiskState.PendingCircuitCloses`; HL CB drain clears pending without on-chain close when `hlLiveStrategiesForCoin(sym)`>1. **Operator-required** (`OKXSpot`,`RobinhoodOptions`): CRITICAL log + notifier per cycle until reset.
- **Kill switch:** `planKillSwitchClose(KillSwitchCloseInputs)`→`KillSwitchClosePlan{OnChainConfirmedFlat}`. New platform = add fields + close/fetcher pair. OKX-spot/RH-options warn but don't block. Auto-reset on confirmed-flat clears virtual state.
- `initial_capital`: only writer `StateDB.SetInitialCapital`; `SaveState` refuses stale overwrites.
- **HL perps stop-loss:** `EffectiveStopLossPct(sc)` priority: ATR/regime → `0` at open → trailing/scalar pct → margin/leverage → `min(MaxDrawdownPct,50)` (seven mutually-exclusive owners, config.go). Fixed ATR `AvgCost ± mult*EntryATR`; regime-aware from `pos.Regime`. Hot-reload: numeric change OK on fresh open; nil↔positive AND scalar↔regime blocked while open. HL caps 1000 triggers/account.
- **HL trailing stop-loss:** `runHyperliquidTrailingStopUpdate` cancels+replaces when trigger moves > `TrailingStopMinMovePct` (0.5%); ATR-mult frozen at open. **SL size cap** `hlSLEffectiveQty` = `min(virtualQty,onChainQty)`.
- HL margin `MarginMode` (`isolated`|`cross`, default `isolated`); `update_leverage` from flat only (HL rejects on open). Multi-strategy same coin: `hyperliquidPeerStrategyErrors` requires shared `margin_mode`+`leverage`, rejects multiple trailing-stop owners (incl. `manual`); sub-account = full independence.
- **HL on-chain protection suppression:** `filterCloseStrategiesForHLOnChainProtection(sc)` strips `tiered_tp_atr`/`tiered_tp_atr_live` for HL **live** perps — prevents limit-fill/close-evaluator race. **Paper never suppressed.** New ATR-tiered evaluator → add to suppression list.
- **SIGHUP hot reload** (`applyHotReloadConfig`): `validateHotReloadCompatible` blocks strategy add/remove, script/args/type/platform/HTFFilter, kill-switch identity, DB path. `validateHotReloadStateCompatible` blocks **while open** `leverage`/`direction`/`invert_signal`/`sl_after`/`*_atr_regime`/`regime_directional_policy` shape changes. When flat, `regime_directional_policy` applies immediately. `invert_signal` (HL perps/manual) flips BUY↔SELL on non-zero (HOLD never) via `applySignalInversion`, before `Direction`.
- **Adding a per-strategy config flag:** (1) field on `StrategyConfig`, (2) main.go `run*Check` appends CLI flag, (3) parse in each Python check script, (4) `InitOptions`+wizard+`generateConfig`. **If *required* by runtime CLI, also append to `probeArgv` in `version_probe.go`** (asymmetric deploy fails at startup).

## Build & Deploy
- **Update:** `bash scripts/update.sh --restart` — atomic: preflight → `pull --ff-only` (or `--rsync-from <src>`, won't clobber `.git/`/config/state DB/venv/binary) → `uv sync` → build → probe → swap (`.prev`) → restart+verify → rollback on timeout. Env `STATUS_PORT`/`ACTIVE_TIMEOUT=30s`/`HEALTH_TIMEOUT=60s`. **Never rebuild Go alone — argv contract requires both sides (Go+Python) at same SHA.** Restart `--restart-mode systemd` (default; falls back to signal) / `signal` (`GO_TRADER_PIDFILE`/`GO_TRADER_RUN_SH`). Batch `--all --restart`.
- **Graceful shutdown:** drains side-effecting subprocesses ≤ `shutdownDrainCap=15s` then SIGKILLs; state-save/notifier-flush/DB-close after via deferred LIFO. Unit `TimeoutStopSec=20`; service-file changes require `daemon-reload`.
- Build (one-off): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (rebuild before smoke-testing). Restart `systemctl restart go-trader`; config-only `kill -HUP $(pgrep go-trader)`; Python picks up next cycle.
- **Startup probe:** binary invokes each unique check script with `--probe-only`; non-zero → log + owner DM + `os.Exit(ExitProbeFailure)` (exit 78, `EX_CONFIG`). `probeFailureScriptMissing` detects `"can't open file"` to distinguish missing-script deploy gaps from argparse mismatches. Both service files set `RestartPreventExitStatus=78` so systemd stays down instead of restart-looping. Failures after update → ensure `shared_scripts/` updated + rebuild.
- **Post-update:** follow `SKILL.md` "Post-Update Agent Protocol" (diff `<running>..HEAD`, classify via Reference table, prompt per item). **Post-deploy smoke:** after a Python-launcher change, `./go-trader --config scheduler/config.json --once`.

## Backtest
- `uv run --no-sync python backtest/run_backtest.py --strategy <n> --symbol BTC/USDT --timeframe 1h --mode single`; `backtest_options.py`/`backtest_theta.py --underlying BTC --since YYYY-MM-DD --capital 10000`.
- **Pairs** (`backtest_pairs.py`, `PairsBacktester`): beta-hedged long/short on rolling z-score of log spread; `--lookback`/`--entry-z`/`--exit-z`/`--beta`. No live path.
- Close registry: `Backtester(open_strategy={...}, close_strategies=[{...}])` mirrors live `StrategyConfig`; `--close-strategy` repeatable; `--config <path> --strategy <id>` (single mode). Max `close_fraction`/bar; entry ATR guard 50%-of-AvgCost; bar-level only.
- Regime: `Backtester(regime_enabled=True, regime_period=14, regime_adx_threshold=20, allowed_regimes=[...])` or `--regime-*`. Entries blocked when `bar_regime ∉ allowed_regimes`; closes always execute. Options backtests don't support regime yet.
- HL-live-only at backtest init (no parity): regime-aware `sl_after` (including regime `tp_atr_fraction`), `regime_directional_policy`. Scalar `sl_after.trail_from_here.tp_atr_fraction` is backtestable. Regime-aware ATR parity: `Backtester` accepts `*_atr_regime` dicts.
- **Look-ahead invariants:** signal at bar N fills at N+1 open; regime gate reads N-1 regime; closes evaluated with closed-bar ATR. Regression `backtest/tests/test_backtester_lookahead.py`.
