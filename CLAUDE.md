# go-trader Project Context

## Environment
- Go 1.26.2; full binary path if not on PATH (`/opt/homebrew/bin/go`).
- Python via `uv run --no-sync python` for dev/backtest/manual CLI; Go subprocess (scheduler) calls `.venv/bin/python3` directly — deterministic after `uv sync`, no systemd PATH config. Worktrees: `uv sync` once per checkout.
- **systemd:** `go-trader.service`, `systemd/go-trader@.service` use `ProtectSystem=strict`; no `PATH`/`UV_CACHE_DIR` injection. `/opt/go-trader/.env` (or per-instance) sets `DISCORD_BOT_TOKEN` + secrets. **#1056** runtime config lives OUT of the deploy tree at `/var/lib/go-trader[/<instance>]/config.json` (writable via `StateDirectory=go-trader[/%i]` — the daemon rewrites config in place, so the directory must stay writable under `ProtectSystem=strict`); `ExecStart … --config <that path>`. `WorkingDirectory` stays the code checkout. `scheduler/config.json` is a transition **symlink** → that path; migrate with `scripts/migrate-config-out-of-tree.sh`.

## Setup
- `uv sync`; copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys. **#1056 (deployments):** see Environment above for the out-of-tree layout — stop the service, run `scripts/migrate-config-out-of-tree.sh` once (refuses while the daemon is live, else a config write in the migrate→restart window clobbers the symlink into an in-tree file), then point `ExecStart --config` + `StateDirectory` at it.
