# go-trader Project Context

## Environment
- Go 1.26.2 — install via your platform's package manager (e.g. `brew install go@1.26` on macOS). If `go` is not in PATH, use the full binary path (e.g. `/opt/homebrew/bin/go` on Apple Silicon).
- Python venv at `.venv/bin/python3` (used by executor.go at runtime); deps via `uv` (`pyproject.toml` / `uv.lock`).
- In git worktrees, `.venv` is NOT copied — use the main repo's venv: `<main-repo>/.venv/bin/python3`.

## Quick Flow
- **New server:** tell your agent `install https://github.com/richkuo/go-trader and init`.

## Setup
- `uv sync` — install Python deps into `.venv`
- Copy `scheduler/config.example.json` → `scheduler/config.json` and fill in API keys

## Repo Structure
- `scheduler/` — Go scheduler (single `package main`); all .go files compile together. Key files:
  - `executor.go` — Python subprocess runner; `pythonSemaphore` caps concurrency at 4, `scriptTimeout=30s`, SIGKILLs process group on timeout.
  - `server.go` — `/status`, `/health`, `/history`; `DefaultStatusPort=8099`, auto-fallback up to 5 ports; precedence: `--status-port` > `cfg.StatusPort` > default.
  - `discord.go` — `discordgo.Session` wrapper; `SendMessage`/`SendDM`/`AskDM`; `FormatCategorySummary`. Category table: `catTableStrategyWidth=18`; `summaryStrategyLabel` aliases (`tiered-atr`→`tatr`, `tiered-pct`→`tpct`) then truncates; cols `Init | Value | PnL | PnL% | DD | Wallet% | Tf | Int | #T | W/L` (DD `%.0f%%`); `Book Sharpe` footer. `collectPositions`: qty `%.3f`; appends `ATR: $<atr>` when `EntryATR>0`, `SL: $<px> (<pct>%)` when `StopLossTriggerPx>0` (no longer gated on OID, #529), `TP1/TP2` from `AvgCost ± EntryATR` for tiered-TP-ATR strategies (`strategyUsesTieredTPATRClose`, #528/#529), `<N>x ($<margin>)` when `Leverage>1` (#485). Trade DMs: `isTradeCloseDetails` (case-insensitive `"close"`) renders partial-close legs as `TRADE CLOSED` (#530); `FormatTradeDM`/`FormatTradeDMPlain` show mode in header; open trade DMs include extras line (Regime / ATR / TP1 / TP2 / SL, #561).
  - `init.go` — `go-trader init` wizard + `--json` mode; `generateConfig(InitOptions)` is pure/testable. Holds `bidirectionalPerpsStrategies`, `knownShortNames`, `defaultSpotStrategies` / `defaultPerpsStrategies` / `defaultFuturesStrategies`.
  - `config.go` / `config_migration.go` — `Config`, `StrategyConfig`; `LoadConfig` infers `Platform` from ID prefix; `CurrentConfigVersion=11`; perps `Leverage` is exchange/risk, `SizingLeverage` controls notional (`cash × sizing_leverage`, defaults to `Leverage`); `MarginPerTradeUSD` (`*float64`, #518) overrides: `notional = min(MarginPerTradeUSD, cash) × leverage`; five HL-perps-only stop fields (`StopLossPct`/`StopLossMarginPct`/`TrailingStopPct`/`TrailingStopATRMult`/`StopLossATRMult`, all `*float64`): mutually exclusive when positive, explicit `0` opts out; sole-owner with all five omitted defaults to `StopLossATRMult=1.0` (#562); `TrailingStopMinMovePct` (debounce, default 0.5%); `MarginMode` (`isolated`|`cross`, default `isolated`); `RegimeConfig{Enabled, Period=14, ADXThreshold=20}` — global detector; `StrategyConfig.AllowedRegimes []string` — entry gate (empty = allow all; rejects unknown labels and type=options; warns when regime disabled globally, #541/#553/#558).
  - `hyperliquid_trailing_stop.go` — synthetic `%` trailing (#501) and ATR-mult distance (#507 on `TrailingStopATRMult` + `Position.EntryATR`): `computeTrailingStopUpdate` walks high-water mark by side and emits cancel/replace when the trigger moves favorably by `TrailingStopMinMovePct`; `runHyperliquidTrailingStopUpdate` invokes `RunHyperliquidUpdateStopLoss` and warns on cancel failure (race against HL 1000-trigger cap).
  - `state.go` / `state_presence.go` / `db.go` — SQLite-only state (`modernc.org/sqlite`); `OpenStateDB`, `SaveStateWithDB`, `LoadStateWithDB`. Tables: `app_state`, `strategies`, `positions`, `closed_positions`, `option_positions`, `closed_option_positions`, `trades`, `portfolio_risk`, `kill_switch_events`, `correlation_snapshot`. `app_state.last_summary_post` (#474) stores wall-clock summary cadence per channel; `trades.position_id` / `positions.position_id` / `option_positions.position_id` (#471) group trade legs into round trips via `idx_trades_strategy_position`. `InsertTrade` writes immediately. `CheckStatePresence` warns on missing DB (`GO_TRADER_ALLOW_MISSING_STATE=1` for first-run). `trades.regime TEXT NOT NULL DEFAULT ''` column (#542) stores the regime label at trade time; `trades.entry_atr REAL` / `trades.stop_loss_trigger_px REAL` (#561) store position context at record time; all three use idempotent migration via duplicate-column guard.
  - `risk.go` / `strategy_interval.go` — `CheckRisk` takes `*PlatformRiskAssist`; `perpsMarginDrawdownInputs` for leverage-aware perps DD. `effectiveStrategyIntervalSeconds` accelerates checks in DD warn band; `WarningSent` repeats warnings every cycle while in band.
  - `kill_switch_close.go` + per-platform closers (`hyperliquid_balance.go`, `okx_close.go`, `robinhood_close.go`, `topstep_close.go`, `robinhood_pending_close.go`, `operator_required_close.go`) — portfolio kill switch + per-strategy CB drains.
  - `*_marks.go`, `deribit.go` — native Go price fetchers (HL `allMids`, OKX swap tickers, Deribit ticker); base URL exposed as `var xxxMainnetURL` for httptest stubs.
  - `regime.go` — `regimeBlocksOpen(allowed, current, posQty)` — blocks new entries when `current` ∉ `allowed` and `posQty==0`; close legs (any existing position quantity) always pass through; `regimeAllowsEntry` is the pure inner helper. `StrategyState.Regime string` persisted in state; `StratStatus.Regime` exposed in `/status` JSON.
  - `sharpe.go`, `failure_alerts.go`, `correlation.go`, `leaderboard.go`, `notifier.go`, `telegram.go`, `updater.go`, `pricer.go` (+ `ibkr_pricer.go`), `shared_wallet.go`, `version.go`, `prompt.go`, `balance.go`, `portfolio.go`, `options.go`, `logger.go`, `tradingview_export.go`, `config_reload.go`.

## Key Patterns
- Run git from repo root (not `scheduler/`). Prefer `go -C scheduler build .` over `cd scheduler && ...` to avoid cwd drift.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` (dispatch), (7) `init.go` (wizard + generateConfig), (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`, `get_real_expiry`, `get_real_strike`, `get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters loaded via `importlib`; class detected by `endswith("ExchangeAdapter")` (one per file). Check scripts must use only public adapter methods.
- **Close registry import:** never `import registry` directly to reach `shared_strategies/close/registry.py` — name collides with the open registry. Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); it loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts always emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of exit code.
- **State locking:** `mu sync.RWMutex` guards `state`. Per-strategy loop has 6 phases: RLock(read inputs) → Lock(CheckRisk) → no lock(subprocess) → Lock(execute signal) → RLock/no lock/Lock(marks) → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)\|mu\.\(R\)\?Unlock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID → platform map: `hl-` HL, `ibkr-` IBKR, `deribit-` Deribit, `ts-` TopStep, `rh-` Robinhood, `okx-` OKX, `luno-` Luno, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`. Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`.
- **Open/close split (#480/#483/#496):** `StrategyConfig.OpenStrategy` overrides entry strategy name; `CloseStrategies` is ordered list of exit evaluators, max `close_fraction` wins; empty falls back to open strategy as implicit close. Composition in `strategy_composition.go` (Go) and `strategy_composition.py` (Python); old configs stay on legacy path. Opted-in scripts emit `open_action`, `close_fraction`, `close_strategy`, and composed `signal`; close runs before opening. Options not yet split. `Regime string` propagated in all 5 result structs and written to `stratState.Regime` each cycle (#541). **Partial-close (#519/#521):** `close_fraction` honored in all live/paper paths; partial close decrements `pos.Quantity`, preserves `InitialQuantity`, skips open-leg fall-through; legacy wrappers pass `0` (full-close). SL cancel/replace suppressed on partial close. **Position-aware close evaluators (#496/#500):** check scripts forward `--position-side`/`--position-avg-cost`/`--position-qty`/`--position-initial-qty`/`--position-entry-atr` (Go: `PositionCtx`). Close functions opt in via kwargs or `**params`; `strip_unsupported_position_context` filters for legacy functions; cache key includes merged params. Reference impl: `tp_at_pct`.
- **Live exec guard:** every dispatch in main.go must use `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** live order helpers (`runHyperliquidExecuteOrder`, `runOKXExecuteOrder`, `runRobinhoodExecuteOrder`, `runTopStepExecuteOrder`) must check the same conditions as `ExecuteXxxSignal` BEFORE spawning the executor — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** `StrategyConfig.AllowShorts` per-strategy opt-in. Strategies emitting `signal=-1` as short entries (`triple_ema_bidir`) must be in `bidirectionalPerpsStrategies` in init.go. `ExecutePerpsSignal(..., allowShorts, ...)` and `PerpsOrderSkipReason(signal, posSide, allowShorts)` must thread `sc.AllowShorts`. Live flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. Threads `pos.AvgCost` from Phase 1 snapshot.
- `dueStrategies` is value-copied from `cfg.Strategies` — mutations don't persist. Update `cfg.Strategies` first.
- Notification features: add methods to `MultiNotifier`, don't access `backends` directly.
- Hyperliquid sys.path conflict: SDK package `hyperliquid` clashes with `platforms/hyperliquid/` — add `platforms/hyperliquid/` directly to sys.path, then `from adapter import HyperliquidExchangeAdapter`.
- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); response uses parallel arrays.
- Fees: `CalculatePlatformSpotFee(platform, value)` (HL 0.035%, RH 0%, BinanceUS 0.1%); `CalculatePlatformFuturesFee(sc, contracts)`.
- Position ownership: `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles with owner only.
- **State is SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). Legacy JSON paths removed. Trades persist immediately via `RecordTrade` → `StateDB.InsertTrade`. `ClosedPositions` flushed atomically by `SaveState`. Leaderboard built on-demand (`BuildLeaderboardMessages`).
- **Trades schema (close legs):** `trades.is_close INTEGER` / `realized_pnl REAL` flag close legs; `trades.regime`, `trades.entry_atr`, `trades.stop_loss_trigger_px` stamped from in-scope `Position` at record time (`stampOpenTradeFromPosition` for open legs, #561). `LifetimeTradeStatsAll(db)` drives `#T`/`W/L`; wins use strict `realized_pnl > 0` (breakeven excluded). Round-trips group by `(strategy_id, position_id)` — partial-close legs collapse into one (#471); NULL/empty `position_id` rows each count as one. `FormatCategorySummary` renders zero when strategy absent — SQLite authoritative, no in-memory fallback (#472). `SaveState` flush INSERT matches `InsertTrade` exactly so retry-queue trades don't lose regime/ATR/SL columns (#561).
- **Live execute fills:** HL/OKX/RH/TopStep `--execute` scripts emit `fillFee` (and per-leg `fillFees`) plus exchange order IDs (`fillOID` / `exchange_order_id`); Go threads these into `Trade.Fees` and `Trade.ExchangeOrderID` BEFORE `RecordTrade`. Python truthy filter (`if oid:`) — never `is not None` — so empty strings and numeric 0 don't land. TopStep precedence: prefer `orderId` even when numeric 0, fall back to `id` only when `orderId` is missing. Tests live in `shared_scripts/test_check_execute_fees.py`.
- **HL kill-switch fill accounting:** Shared-coin fills split by **virtual quantity at snapshot time** (not capital weight): snapshot taken under RLock BEFORE the close mutates state. `hyperliquidKillSwitchFillShare` fails closed (returns `0, 0`) when strategy isn't a peer or has no virtual qty — never claims the entire portfolio fill.
- **Trade.Side on close legs:** close trades record buy/sell semantics, not the literal string `"close"`. Use `closeTradeSide(positionSide)` (perps/spot — `"short"` → `"buy"`, else `"sell"`) and `optionCloseTradeSide(action)` (options — `"sell"` → `"buy"`, else `"sell"`). Applies in `forceCloseAllPositions`, `recordPerpsStopLossClose`, `executeOptionClose`, and `CheckThetaHarvest`. Default-to-`"sell"` preserves legacy behavior for missing/unknown sides.
- **TradingView export:** if the user says "export data to TradingView", first ask which strategy IDs to export or whether to export all strategies. Use `./go-trader export tradingview --strategy <id> --output <file>` for one or more selected strategies, or `./go-trader export tradingview --all --output <file>`. Exports are sourced from SQLite trades and may require `tradingview_export.symbol_overrides` for platforms/symbols without a safe built-in TradingView mapping.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting any operator-facing or test-asserted output. Go map iteration is randomized.
- `cfg.Discord.Channels` / `Telegram.Channels` / `DMChannels` are `map[string]string`; keys: `spot`, `options`, `<platform>`, `<platform>-paper`. `cfg.Discord.OwnerID` from `DISCORD_OWNER_ID` env var (priority over config).
- `cfg.SummaryFrequency map[string]string`: Go duration / alias (`hourly`, `daily`, `every`/`per_check`/`always`) / empty (legacy). `ShouldPostSummary(freq, continuous, hasTrades, lastPost, now)` — wall-clock throttling (#474), not cycle-count; `hasTrades=true` always forces a post; per-channel `lastPost` tracked in `lastSummaryPost map[string]time.Time` (single-writer in main loop, mirrored into `state.LastSummaryPost` during save) and persisted in `app_state.last_summary_post` so cadence survives restart/auto-update. Use `cloneTimeMap` when copying between the local map and `AppState`.
- `cfg.Correlation` (`Enabled=false`, `MaxConcentrationPct=60`, `MaxSameDirectionPct=75`); `cfg.AutoUpdate`: `off|daily|heartbeat`.
- **Regime detection (#539–#558):** `shared_tools/regime.py` emits 3-state labels (`trending_up`, `trending_down`, `ranging`) via ADX+DI; canonical set in `validRegimeLabels` (`config.go`). All 5 standard check scripts inject label; `check_options.py` emits `"regime": null` (deferred). `"regime"` is in `POSITION_CONTEXT_PARAM_KEYS` so `strip_unsupported_position_context` drops it for strategies without a `regime=` kwarg. Go appends `--regime-enabled`/`--regime-period`/`--regime-adx-threshold` via `appendRegimeArgs` (#558); absent → scripts emit `regime=""`. Entry gate `regimeBlocksOpen(allowed, current, posQty)` — close legs (posQty>0) always pass. `AllowedRegimes` SIGHUP-compatible; `cfg.Regime` block requires restart.
- **Strategy registries (#511):** open implementations live once under `shared_strategies/open/`, source of truth `shared_strategies/open/registry.py`; spot/futures shims build fresh `STRATEGY_REGISTRY` via `build_registry()`. Close evaluators live under `shared_strategies/close/` and register in `shared_strategies/close/registry.py` with `evaluate(position, market, params)`. New open strategy checklist: (1) register in `shared_strategies/open/registry.py` + `PLATFORM_ORDER`, (2) `knownShortNames` + defaults in `init.go`, (3) `DEFAULT_PARAM_RANGES` in `backtest/optimizer.py`. Use `variants={"futures":{...}}` rather than duplicating. Before refactoring registry: snapshot `--list-json` output and `diff` after — Go `discoverStrategies` depends on byte-identical output.
- `apply_strategy(name, df, params)` — config params merged UNDER runtime params (runtime priority).
- `check_strategy.py` uses manual `sys.argv` parsing — filter `--`-prefixed args before indexing. Other check scripts use argparse.
- `StrategyConfig.HTFFilter` — Go appends `--htf-filter`; not applied to options or `delta_neutral_funding`.
- `delta_neutral_funding` is perps-only — function in `spot/strategies.py` but registered only for futures.
- Perps vs futures at Position level: both `Multiplier > 0`; only perps `Leverage > 0`. Use both fields for leverage-aware metrics.
- Per-strategy DD: spot/options/futures use peak-relative; perps uses `unrealized_loss / deployed_margin` when positions open. Margin uses exchange `Leverage`, not `SizingLeverage` (#497). `RecordTradeResult` only updates `DailyPnL` and `ConsecutiveLosses` — lifetime trade counts come from SQLite, not `RiskState` (#472).
- **Test pattern:** extract pure helpers (`perpsLiveOrderSize`, `*OrderSkipReason`, `parseXxxCloseOutput`) from subprocess-spawning wrappers — Go CI lacks `.venv/bin/python3` so any test calling `RunPythonScript` / `RunHyperliquidExecute` / `RunHyperliquidClose` fails CI.
- Per-strategy circuit-breaker pending close: `RiskState.PendingCircuitCloses map[string]*PendingCircuitClose` keyed by `PlatformPendingClose{Hyperliquid,OKX,Robinhood,TopStep,OKXSpot,RobinhoodOptions}`. Use `setPendingCircuitClose`/`clearPendingCircuitClose`/`getPendingCircuitClose` accessors. `CheckRisk` takes `*PlatformRiskAssist`. RH enqueue runs only from drain's stuck-CB recovery (avoids TOTP cost). **HL perps CB drain (#515):** when `hlLiveStrategiesForCoin(sym)` reports more than one live strategy on the same coin/wallet, the Hyperliquid drain clears that strategy's pending close without issuing an on-chain close (prevents accidental full-position closes shared with peers) — `[INFO] hl-circuit-close: ... shares the wallet position with peers ...`.
- **Operator-required CBs** (`OKXSpot`, `RobinhoodOptions`): `OperatorRequired=true`; `drainOperatorRequiredPendingCloses` emits one CRITICAL log + notifier per cycle until CB resets.
- **Kill switch:** `planKillSwitchClose(KillSwitchCloseInputs)` returns pure `KillSwitchClosePlan` with `OnChainConfirmedFlat`. Adding new platform = add fields + close/fetcher pair, not signature change. OKX-spot / RH-options surface as warnings, don't block confirmed-flat. Per-platform timeouts (`HLCloseTimeout`, `RHCloseTimeout`, ...). Auto-reset on confirmed-flat: clears virtual state, resumes trading next cycle.
- **`initial_capital` immutable:** `SaveState` refuses stale in-memory overwrites. ONLY write path: `StateDB.SetInitialCapital(strategyID, value)`.
- SQLite column rename migration: use `PRAGMA table_info` to detect three states (neither, legacy-only, new-only); idempotent. `UnmarshalPendingCircuitClosesJSON` accepts both new map and legacy `{"coins":[...]}` shapes.
- **HL perps stop-loss:** `EffectiveStopLossPct(sc)` priority: `StopLossATRMult` or `TrailingStopATRMult` positive → returns `0` at open (arms next cycle once `Position.EntryATR` set) → `TrailingStopPct` → `StopLossPct` → `StopLossMarginPct/Leverage` → `min(MaxDrawdownPct,50)` sole-owner fallback. Explicit `0` passes through (e.g. `stop_loss_atr_mult: 0` + `stop_loss_pct: 2` still arms). At most one positive owner across all five fields. `LoadConfig` calls `normalizeHyperliquidPeerStopLosses` for same-coin peers; defaults sole-owner with all five omitted to `StopLossATRMult=1.0` (#562). **Fixed ATR (`StopLossATRMult`)**: trigger `AvgCost ± mult*EntryATR`, armed once via `hyperliquidArmFixedATRStopLossLive` (live) / `runHyperliquidFixedATRStopLossPaper` (paper), never updated. Hot-reload: numeric change on next fresh open; nil↔positive blocked while open. Missing-EntryATR alerts share `atrMultMissingEntryATRWarned` with trailing variant. OID on `Position.StopLossOID`; HL caps trigger orders at 1000/account (#481).
- **HL trailing stop-loss (#501/#502/#507/#532):** Live — `runHyperliquidTrailingStopUpdate` walks `StopLossHighWaterPx`, cancels+replaces when trigger moves > `TrailingStopMinMovePct` (default 0.5%); cancel failures NOTIFIER-alert (1000-trigger cap racing). ATR-mult distance `TrailingStopATRMult * EntryATR / AvgCost` frozen at open; hot-reload can change mult when flat; nil↔positive blocked while open; peer validation matches exchange `Leverage` not `SizingLeverage`. Paper (#532) — `runHyperliquidTrailingStopPaper` reuses the same helpers, books close via `recordPerpsStopLossClose(..., "trailing_stop_loss_paper", ...)`; strategy-scoped (peers retain virtual exposure). When extending live-only features to paper, ungate diagnostics from `hyperliquidIsLive` so throttle maps are shared.
- HL perps margin mode: `StrategyConfig.MarginMode` (`isolated`|`cross`, default `isolated`, #486). Scheduler calls `update_leverage(leverage, coin, is_cross=...)` from flat (`posQty==0`) before opening; HL rejects mode/leverage changes on an open position. Hot-reload mirrors the `Leverage` envelope — change allowed when flat, rejected with positions open.
- Multi-strategy HL perps on same coin (#491/#494/#507): `LoadConfig` coerces omitted stop/trailing fields to explicit `0` for peers (`normalizeHyperliquidPeerStopLosses`) then validates shared `margin_mode`+`leverage` and at-most-one positive stop owner (`hyperliquidPeerStrategyErrors`). `check_hyperliquid.py` skips `update_leverage` when current state already matches. `validateHotReloadCompatible` mirrors the peer check. Sub-account isolation is the only correct path for full independence.
- **SIGHUP hot reload:** `applyHotReloadConfig` (config_reload.go) re-applies a subset of config without restart. `validateHotReloadCompatible` blocks shape changes (strategy add/remove, script/args/type/platform/HTFFilter, AllowShorts mid-run, kill-switch identity, DB path); `validateHotReloadStateCompatible` blocks per-strategy `leverage` changes when positions are open. Notifier reload re-routes Discord/Telegram channels in place; guard new backends behind nil-checks.
- Drain-failure alerts: `shouldNotifyDrainFailure(key, throttleMap)` + `formatDrainFailureAlert` (failure_alerts.go); throttles per (strategy, platform, symbol, direction).
- Sharpe: per-strategy + book-level annualized; rendered in leaderboard col + `Book Sharpe` footer. `RiskFreeRate` baseline from config (explicit 0 respected). Strategy-interval speedup: when DD > `warn_threshold_pct`, returns `strategyDrawdownFastIntervalSeconds`.
- Adding a per-strategy config flag: (1) field on `StrategyConfig`, (2) main.go `run*Check` appends CLI flag, (3) parse in each Python check script, (4) `InitOptions` + wizard + `generateConfig`.
- Test helpers: name with platform/feature prefix (`stubHLLiveCloser`, not `stubCloser`) — `package main` is shared across `*_test.go`.

## Pull Requests
- Reference related issue with `Closes #<N>` in body.
- In GitHub comments avoid `#N` for list items (auto-links to issues/PRs); use `1.` instead.
- Fetch latest bot review: `gh api repos/richkuo/go-trader/issues/<N>/comments --jq '[.[] | select(.user.login=="codex[bot]" or .user.login=="claude[bot]")] | last | .body'` (top-level summary lives on issues endpoint, not pulls).
- Before merging long-running PR: `git fetch origin main && git diff origin/main..HEAD -- <paths>` to catch silent reverts; rebase if unexpected deletions.
- Replace default `🤖 Generated with [Claude Code]...` footer with metadata (model + effort + harness), e.g. `LLM: Claude Sonnet 4.6 (1M) | high | Harness: anthropics/claude-code-action@v1`. No `Co-Authored-By` trailer.

### PR review format (`@claude review`)
When invoked to review a PR, the top-level review comment MUST take exactly one of two shapes — no preamble, no closing remarks:
1. **Approve:** a single line starting with `LGTM — ` followed by a one-sentence rationale (what was checked + why it's safe to merge).
2. **Changes requested:** a numbered list (`1.`, `2.`, …) where each item is one sentence naming the concrete change, with `file:line` when applicable. Order by severity (blockers first). No "nice to haves" mixed in — if it's not blocking, leave it out or put it under a final `Optional:` line.
Inline `pull_request_review_comment` threads are exempt from this format; this rule governs only the top-level review summary.
