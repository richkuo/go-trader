# go-trader Project Context

## Environment
- Go 1.26.2; full binary path if not on PATH (`/opt/homebrew/bin/go`).
- Python via `uv run --no-sync python` for dev/backtest/manual CLI; Go subprocess (scheduler) calls `.venv/bin/python3` directly — deterministic after `uv sync`, no systemd PATH config. Worktrees: `uv sync` once per checkout.
- **systemd:** `go-trader.service`, `systemd/go-trader@.service` use `ProtectSystem=strict`; no `PATH`/`UV_CACHE_DIR` injection. `/opt/go-trader/.env` (or per-instance) sets `DISCORD_BOT_TOKEN` + secrets. **#1056** config lives OUT of the deploy tree at `/var/lib/go-trader[/<instance>]/config.json` (`StateDirectory=go-trader[/%i]`, writable under `ProtectSystem=strict`); `ExecStart … --config <that path>`. `WorkingDirectory` stays the code checkout. `scheduler/config.json` is a transition **symlink**; migrate with `scripts/migrate-config-out-of-tree.sh`.

## Setup
- `uv sync`; copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys. **#1056 (deployments):** see Environment above — stop the service, run `scripts/migrate-config-out-of-tree.sh` once (refuses while the daemon is live, else a config write in the migrate→restart window clobbers the symlink into an in-tree file), then point `ExecStart --config` + `StateDirectory` at it.

## Decision Priorities
- **Always pursue the absolute best technical solution — full stop.** Cost, compute, resources, time-to-implement, effort, manpower, tests, and code volume are NOT design constraints and must never narrow the option space: choose the best solution as if these were unlimited, then update/rewrite tests after the feature is done. These govern *quality*, not *scope*; don't override the branch+PR workflow, "verify issue claims against code", or destructive-action safety rules.
- **Never give time, duration, or effort estimates** ("2–4 days", "low effort", headcount/manpower) in responses, PRs, issues, or commits. Describe complexity only in terms of scope and risk.

## Build & Deploy
- **Update:** `bash scripts/update.sh --restart` — atomic: preflight → `pull --ff-only` (or `--rsync-from <src>`) → `uv sync` → build → probe → swap (`.prev`) → restart+verify → rollback on timeout. **Never rebuild Go alone — argv contract requires both sides (Go+Python) at same SHA.** Batch `--all --restart` auto-discovers deployments via `discover_deployment_dirs_from_systemd`. **Detail → ARCHITECTURE.md.**
- **Graceful shutdown:** drains side-effecting subprocesses ≤ `shutdownDrainCap=15s` then SIGKILLs; state-save/notifier-flush/DB-close after via deferred LIFO. Unit `TimeoutStopSec=20`; service-file changes require `daemon-reload`.
- Build (one-off): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (rebuild before smoke-testing). Restart `systemctl restart go-trader`; config-only `kill -HUP $(pgrep go-trader)`; Python picks up next cycle.
- **Startup probe:** binary invokes each unique check script with `--probe-only`; non-zero → log + owner DM + `os.Exit(ExitProbeFailure)` (exit 78, `EX_CONFIG`). `probeFailureScriptMissing` detects `"can't open file"` to distinguish missing-script deploy gaps from argparse mismatches. Both service files set `RestartPreventExitStatus=78`. Failures after update → ensure `shared_scripts/` updated + rebuild.
