Needs Fixing / Recommended Optional add **Invariant:** + **Must survive:** (1–3 adversarial cases). **`### Create Follow-up Issue` is last-resort — strongly prefer keeping work in the current PR;** a new issue needs BOTH genuine scope-separateness AND infeasibility of folding into this PR (substantial independent scope / own design decision / would bloat the diff).
