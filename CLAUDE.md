## Decision Priorities
- **Pick the best technical solution.** Don't down-scope to avoid test churn, large diffs, or long implementation time. Those are not constraints on design quality.
- **Tests are not a limiting factor for design.** If the right implementation breaks existing tests, that is acceptable — update or rewrite the tests after the feature is complete. Do not contort a design to keep tests passing.
- **Code volume is not a constraint.** Don't shrink scope to minimize lines changed. If 2000 lines is the right answer, write 2000 lines.
- **Time-to-implement is not a constraint.** Don't pick a faster-but-worse approach to "save time." Pick the correct approach.
- These rules govern *quality of the chosen solution*, not *scope of the task*. They don't override anti-over-engineering guidance, the branch+PR workflow, the "verify issue claims against code" rule, or destructive-action safety rules.

## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler &&`.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go` wizard + `generateConfig`, (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`/`get_real_expiry`/`get_real_strike`/`get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters via `importlib`; class detected by `endswith("ExchangeAdapter")` (one/file). Check scripts use public methods only.
- **Close registry import:** never `import registry` directly (collides with open registry). Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of code.
- **State locking:** `mu sync.RWMutex` guards `state`. Per-strategy 6-phase loop: RLock(read) → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → RLock/no-lock/Lock(marks) → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)\|mu\.\(R\)\?Unlock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map: `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`, `manual`. Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`. **Manual** has no script/interval — `LoadConfig` auto-fills `script`/`args[0]="hold"`/`interval_seconds`/SL@1.5×ATR + `tiered_tp_atr_live`; scheduler runs close-eval, enqueues into `pending_manual_actions`. `CheckRisk` skipped. May share a coin with manual/HL perps peers — owner guards prevent cross-strategy mutation; full-close uses `shouldCloseFullPosition` + `extraCancelOIDs`.
- **Open/close split:** `OpenStrategy` (StrategyRef) overrides entry name; `CloseStrategies` ordered, max `close_fraction` wins; empty → open-as-close. Composition in `strategy_composition.{go,py}`. Go emits `--strategy-refs <json>`; Python `parse_strategy_refs_arg` + `evaluate_open_close`. Duplicate close-ref names: last wins. Close runs before open. Options not split. `Regime` propagated. **Partial-close:** `close_fraction` decrements `pos.Quantity`, preserves `InitialQuantity`; SL cancel/replace suppressed. **Position-aware close evaluators:** scripts forward `--position-{side,avg-cost,qty,initial-qty,entry-atr,regime}` (`PositionCtx`); close fns opt in via explicit kwargs. `strip_unsupported_position_context` filters framework keys — **VAR_KEYWORD wrappers don't short-circuit (#721):** `def *_strategy(df, **params)` must name framework kwargs explicitly to forward them. Cache key includes merged params.
- **Live exec guard:** every dispatch in main.go uses `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** `runHyperliquidExecuteOrder` / `runOKXExecuteOrder` / `runRobinhoodExecuteOrder` / `runTopStepExecuteOrder` must check the same conditions as `ExecuteXxxSignal` BEFORE spawning — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** per-strategy `Direction` enum (see config bullet). Short-entry strategies (`triple_ema_bidir`, `bear_pullback_st`, `vwap_rejection_st`, `donchian_breakout`, `chart_pattern`, `liquidity_sweeps`) must be in `bidirectionalPerpsStrategies` (init.go). `ExecutePerpsSignalWithLeverage` + `PerpsOrderSkipReason` thread the resolved direction. Live flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. Threads `pos.AvgCost` from Phase 1 snapshot. `ValidatePerpsDirectionConfig` flags both gap classes.
- `dueStrategies` is value-copied from `cfg.Strategies` — mutations don't persist; update `cfg.Strategies` first.
- Notification features: add methods to `MultiNotifier`; don't access `backends`.
- HL sys.path: SDK `hyperliquid` clashes with `platforms/hyperliquid/` — add `platforms/hyperliquid/` directly to sys.path, then `from adapter import HyperliquidExchangeAdapter`.
- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); parallel arrays.
- Fees: `CalculatePlatformSpotFee` (HL 0.035%, RH 0%, BinanceUS 0.1%); `CalculatePlatformFuturesFee`.
- Position ownership: `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles owner only. **Shared-coin reconcile:** `reconcileHyperliquidAccountPositions` non-destructive for ambiguous gaps; three unambiguous-close detectors:
  1. **Full external close** — on-chain qty ≈ 0 with non-zero virtual → close all peers (SL owner via `hl_sync_stop_loss`; others via `recordPerpsExternalClose`, mark-based PnL).
  2. **SL-owner partial close** — sole SL owner; on-chain residual matches signed sum of non-owner peers → close owner only.
  3. **TP partial fill** — `delta>0`, `hyperliquidSharedPartialCloseDrift` reports one closeable side/qty AND exactly one same-side strategy has a cleared TP tier (`hyperliquidHasClearedTPTier`) → `recordPerpsExternalPartialCloseWithFillFee` + shrink virtual qty. Multiple candidates → leave gap visible.
  Close paths use real HL fee via `buildCachedHyperliquidReconcileFillResolver`; failure → modeled. When `useFillFee && FilledQty < virtualQty`, override `pos.Quantity` with actual fill size before PnL booking. **Scope:** `hlReconcileAll`/`Due` via `isHLLiveReconcilable` (HL+live+perps|manual); `hlLiveAll`/`Due` perps-only.
  **Owner DM on TP/SL fill:** detectors append `ProtectionFillAlert` (`<FillType> filled (oid=<id>)` headline) to `pendingAlerts` under `mu`; deferred `notifyProtectionFill` flushes after unlock (Discord HTTP must not stall RLocks). TP tier from `hyperliquidClearedTPTier`. Toggle: `notify_tp_sl_fills` (`*bool`, nil→on).
  **Sole-owner TP attribution (non-shared):** `tryBookSoleOwnerTPFill` (via `reconcileHyperliquidPositionsForStrategy`) books partial drift / final-tier flatten at `HLFillLookup.Px` (or configured TP price) + `ProtectionFillAlert`. **Full-close requires ALL `TPOIDs[i]==0`** — any active tier defers to SL branch.
- **State is SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). Trades persist via `RecordTrade` → `StateDB.InsertTrade`. `ClosedPositions` flushed by `SaveState`. Leaderboard on-demand (`BuildLeaderboardMessages`).
- **Trades schema:** `is_close`/`realized_pnl` flag close legs; close-leg `regime`/`entry_atr`/`stop_loss_trigger_px` stamped via `stampOpenTradeFromPosition`. **Open-trade snapshot:** `recordPositionOpen(s,sc,trade,pos)` single INSERT with full protection snapshot (`stampPositionProtectionSnapshot`+`copyPositionOpenSnapshotToTrade`). `Execute{Spot,Perps,Futures}*DeferredOpen` return `SignalExecutionResult`; main loop runs `runHyperliquidProtectionSync` between them and `recordPositionOpen` so SL/TP OIDs land same-cycle. `LifetimeTradeStatsAll`: `#T` counts opens; W/L round-trip-grouped by `(strategy_id, position_id)`, wins strict `realized_pnl > 0`.
- **Live execute fills:** HL/OKX/RH/TopStep `--execute` scripts emit `fillFee`+exchange order IDs; Go threads into `Trade.Fees`/`Trade.ExchangeOrderID` BEFORE `RecordTrade` (TopStep prefers `orderId` even when 0). **HL real fee:** `lookup_fill_fee_by_oid` aggregates fee+closedPnl across partial fills; failure → modeled `CalculatePlatformSpotFee`. Historical `exchange_fee=0` rows correctable via `go-trader backfill hl-fees`. **HL `closedPnl` is gross of fees** (`HLFillLookup.ClosedPnLGross`) — never wire into `Trade.RealizedPnL`; PnL computed locally in `bookPerpsPartialCloseWithFillFee`.
- **HL kill-switch fill accounting:** shared-coin fills split by **virtual qty at snapshot time** (not capital weight); snapshot under RLock BEFORE close mutates state. `hyperliquidKillSwitchFillShare` fails closed (`0,0`) when strategy isn't a peer or has no virtual qty.
- **Trade.Side on close legs:** records buy/sell semantics, not `"close"`. `closeTradeSide(positionSide)` (perps/spot — `"short"`→`"buy"`, else `"sell"`); `optionCloseTradeSide(action)` (options — `"sell"`→`"buy"`, else `"sell"`). Applies in `forceCloseAllPositions`/`recordPerpsStopLossClose`/`executeOptionClose`/`CheckThetaHarvest`.
- **Close-leg trade-alert `Source:` line:** close legs add `Source: <exchange SL|exchange TP{n}|close-strategy exit|external|circuit breaker|paper SL|paper trailing SL|trailing SL>` from `Trade.Details` prefix. `recordPerpsStopLossClose` writes prefix via `stopLossCloseDetailsPrefix(reason)`; `tradeAlertCloseSource` classifies. New reason → both helpers need a branch (paper-trailing substring check precedes plain trailing).
- **TradingView export:** when user says "export data to TradingView", first ask which strategy IDs (or all). Use `./go-trader export tradingview --strategy <id> --output <file>` or `--all --output <file>`. Sourced from SQLite trades; may require `tradingview_export.symbol_overrides` for unmapped platforms/symbols.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting operator-facing or test-asserted output — Go map iteration is randomized.
- `cfg.Discord.Channels` / `Telegram.Channels` / `DMChannels` are `map[string]string`; keys: `spot`, `options`, `<platform>`, `<platform>-paper`. `cfg.Discord.OwnerID` from `DISCORD_OWNER_ID` env (priority over config).

## Build & Deploy
- **Update (operator + agents):** `bash scripts/update.sh --restart`. Atomic flow: preflight (hard-fails on missing `scheduler/config.json` / dirty build inputs) → `pull --ff-only` → `uv sync` → build `go-trader.new` → probe → atomic swap (`.prev` retention) → `restart`+verify (`systemctl is-active` + `/health` version match AND `MainPID` ≠ pre-restart PID; honors custom systemd unit name #727). On timeout/mismatch `do_rollback` resets git to `pre_pull_sha`, re-syncs uv if HEAD advanced, restores `.prev`. Env: `STATUS_PORT`, `ACTIVE_TIMEOUT=30s`, `HEALTH_TIMEOUT=60s`. `applyUpgrade` calls same script. **Do not** rebuild Go in isolation — argv contract needs both sides at same SHA.
- **Graceful shutdown:** `systemctl restart` drains side-effecting subprocesses up to `shutdownDrainCap=15s` then SIGKILLs; state-save / notifier flush / DB close run after. Unit `TimeoutStopSec=20`. Journal: `[shutdown] draining` / `State saved` / `Complete`. `/health` 503 while draining. Service-file change requires `daemon-reload`.
- Build (one-off): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (without `-ldflags` summary shows `(dev)`). Rebuild before smoke-testing.
- Restart: `systemctl restart go-trader` (service-file changes: `daemon-reload` first). Config-only: `kill -HUP $(pgrep go-trader)`. Python changes pick up next cycle.
- **Startup compatibility probe:** binary invokes each unique check script with `--probe-only`; non-zero → log + owner DM + `os.Exit(1)`. Startup failure post-update → ensure `git pull` updated `shared_scripts/` and rebuild.
