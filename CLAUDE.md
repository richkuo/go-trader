# go-trader Project Context

## Environment
- Go 1.26.2 (`brew install go@1.26` on macOS); use full binary path if not on PATH (e.g. `/opt/homebrew/bin/go`).
- Python runs through `uv run --no-sync python` for dev/backtest/manual CLI use; Go subprocess calls (scheduler) use `.venv/bin/python3` directly — deterministic relative path after `uv sync`, no PATH config needed in systemd.
- Git worktrees work without a copied virtualenv; run `uv sync` once per checkout.
- **systemd:** `go-trader.service` and `systemd/go-trader@.service` use `ProtectSystem=strict`; no `PATH`/`UV_CACHE_DIR` env injection needed since scheduler uses `.venv/bin/python3`. `/opt/go-trader/.env` (or per-instance `.env`) can set `DISCORD_BOT_TOKEN` and other secrets.

## Setup
- `uv sync` — install Python deps
- Copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys
- New server: `install https://github.com/richkuo/go-trader and init`

## Decision Priorities
- **Pick the best technical solution.** Don't down-scope to avoid test churn, large diffs, or long implementation time. Those are not constraints on design quality.
- **Tests are not a limiting factor for design.** If the right implementation breaks existing tests, that is acceptable — update or rewrite the tests after the feature is complete. Do not contort a design to keep tests passing.
- **Code volume is not a constraint.** Don't shrink scope to minimize lines changed. If 2000 lines is the right answer, write 2000 lines.
- **Time-to-implement is not a constraint.** Don't pick a faster-but-worse approach to "save time." Pick the correct approach.
- These rules govern *quality of the chosen solution*, not *scope of the task*. They don't override the branch+PR workflow, the "verify issue claims against code" rule, or destructive-action safety rules.

## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler &&`.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go` wizard + `generateConfig`, (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`/`get_real_expiry`/`get_real_strike`/`get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters via `importlib`; class detected by `endswith("ExchangeAdapter")` (one/file). Check scripts use public methods only.
- **Close registry import:** never `import registry` directly (collides with open registry). Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of code.
- **State locking:** `mu sync.RWMutex`. 6-phase loop: RLock → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → marks → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map: `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`, `manual`. Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`. **Manual** has no script/interval — `LoadConfig` auto-fills `script`/`args[0]="hold"`/`interval_seconds`/SL@1.5×ATR + `tiered_tp_atr_live`; scheduler runs close-eval, enqueues into `pending_manual_actions`. `CheckRisk` skipped. May share a coin with manual/HL perps peers — owner guards prevent cross-strategy mutation; full-close uses `shouldCloseFullPosition` + `extraCancelOIDs`.
- **Open/close split:** `OpenStrategy` overrides entry; `CloseStrategies` ordered, max `close_fraction` wins; empty → open-as-close. `strategy_composition.{go,py}`. Go emits `--strategy-refs <json>`; Python `parse_strategy_refs_arg` + `evaluate_open_close`. Duplicate close-ref names: last wins. Close before open. Options not split. `Regime` propagated. **Partial-close:** `close_fraction` decrements `pos.Quantity`, preserves `InitialQuantity`; SL cancel/replace suppressed. **Position-aware close evaluators:** `--position-{side,avg-cost,qty,initial-qty,entry-atr,regime}` (`PositionCtx`); `strip_unsupported_position_context` filters keys. `def *_strategy(df, **params)` must name framework kwargs explicitly (VAR_KEYWORD wrappers don't short-circuit). Cache key includes merged params.
- **Live exec guard:** every dispatch in main.go uses `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** `runHyperliquidExecuteOrder` / `runOKXExecuteOrder` / `runRobinhoodExecuteOrder` / `runTopStepExecuteOrder` must check the same conditions as `ExecuteXxxSignal` BEFORE spawning — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** `Direction` enum (`"long"`|`"short"`|`"both"`). Short-entry strategies (`triple_ema_bidir`, `bear_pullback_st`, `vwap_rejection_st`, `donchian_breakout`, `chart_pattern`, `liquidity_sweeps`) must be in `bidirectionalPerpsStrategies` (init.go). `ExecutePerpsSignalWithLeverage`+`PerpsOrderSkipReason` thread direction. Flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. `ValidatePerpsDirectionConfig` (#783) compares open side to `EffectiveDirectionForPosition` (stamped regime when present); with `regime_directional_policy` and empty stamp, `policyAllowsPositionSide` suppresses false positives.

- Notification features: add methods to `MultiNotifier`; don't access `backends`.

- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); parallel arrays.

## Build & Deploy
- **Update (operator + agents):** `bash scripts/update.sh --restart`. Atomic flow: preflight → `pull --ff-only` → `uv sync` → build `go-trader.new` → probe → atomic swap (`.prev`) → restart+verify → rollback on timeout. Env: `STATUS_PORT`, `ACTIVE_TIMEOUT=30s`, `HEALTH_TIMEOUT=60s`. **Do not** rebuild Go alone — argv contract requires both sides at same SHA. **Go binary:** PATH `go` else `/opt/homebrew/bin/go` else `/usr/local/go/bin/go`. **Restart modes:** `--restart-mode systemd` (default; `systemctl restart <unit>`, `MainPID` verify) or `--restart-mode signal` (SIGTERM old pid from `GO_TRADER_PIDFILE`, respawn via `GO_TRADER_RUN_SH`). Generate `run.sh`: `bash scripts/create-run-sh.sh`. **Batch:** `--all --restart [--update-all-root <dir>]`. Signal-mode env: `GO_TRADER_RUN_SH`, `GO_TRADER_PIDFILE`, `GO_TRADER_SIGNAL_LOG`, `GO_TRADER_UPDATE_ALL_ROOT`.
- **Graceful shutdown:** drains side-effecting subprocesses up to `shutdownDrainCap=15s` then SIGKILLs; state-save/notifier-flush/DB-close run after. Unit `TimeoutStopSec=20`. Journal: `[shutdown] draining` / `State saved` / `Complete`. `/health` 503 while draining. Service-file changes require `daemon-reload`.
- Build (one-off): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (without `-ldflags` summary shows `(dev)`). Rebuild before smoke-testing.
- Restart: `systemctl restart go-trader` (service-file changes: `daemon-reload` first). Config-only: `kill -HUP $(pgrep go-trader)`. Python changes pick up next cycle.
- **Startup probe:** binary invokes each unique check script with `--probe-only`; non-zero → log + owner DM + `os.Exit(1)`. Post-update probe failures → ensure `shared_scripts/` updated + rebuild.
- **Post-update protocol:** follow `SKILL.md` "Post-Update Agent Protocol". Diff `<running>..HEAD`, classify via SKILL.md Reference table, prompt per item.
- **Post-deploy smoke:** after a Python-launcher change, run `./go-trader --config scheduler/config.json --once` to confirm probe + slow scripts finish within timeouts.
