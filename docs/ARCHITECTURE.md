# go-trader Architecture Reference

> **Companion to `CLAUDE.md`.** CLAUDE.md is loaded into every session and carries the
> *guardrails* — the imperative "do X here / NEVER Y / lock order / wire both sides / fails
> closed" rules. This file carries the *descriptive depth* those guardrails compress: what each
> file owns, full symbol/struct/config inventories, per-feature mechanics, and the issue
> rationale. It is **not** auto-loaded — **read the relevant section here before editing that
> subsystem.** If you add a new guardrail, it goes in CLAUDE.md; the explanation goes here.

## `scheduler/` file-ownership map

`scheduler/` is one Go `package main`.

- `executor.go`/`shutdown.go` — Python subprocess runner (`pythonSemaphore=4`, `scriptTimeout=30s`); drain waits `shutdownDrainCap=15s` → SIGKILL. **New side-effecting wrapper → `runPythonSideEffect`, NEVER `runPython`.**
- `server.go`/`ui_server.go`/`ui_tuner.go`/`ui_candles.go`/`ui_reports.go`/`static/ui/*` — HTTP `/status`,`/health`,`/history`,`/dashboard`,`/reports`(+`/reports/strategy-audit`, #956),`/api/strategies/...`; `DefaultStatusPort=8099` (+5 fallback), **loopback only**, `/health` 503 while draining, `version` for update.sh. **Lock order `mu → strategiesMu`** (reusing `mu` deadlocks SIGHUP). Dashboard API endpoints per strategy: `candles`,`trades`,`status`,`equity`,`config`(GET+POST),`simulate`(POST). `UIEquityPoint{T,V}`, `uiEquityLookbackLimit=500`; `UIStrategyOverview` at `/api/strategies/overview`. **Tuner** (`ui_tuner.go`): `UIEditableField`, `UIStrategyConfigResponse`, `UISimulateRequest`/`Response`, `UIApplyConfigRequest`/`Response`; `applyStrategyConfigPatch` requires `config_version>=13`; `requireMutatingAPIAuth` enforces `status_token` for POST `/config`; `requireSameOrigin` same-origin; `configWriteMu` serializes writes; `SetConfigContext(configPath, regime)` wires tuner at startup.
- `config.go`/`config_migration.go` — `CurrentConfigVersion=15`; `LoadConfig` infers `Platform` from ID prefix; `LoadConfigForProbe` skips HL cred checks. **Seven mutually-exclusive HL stop fields** (5 scalar pct/ATR + `StopLossATRRegime`/`TrailingStopATRRegime`); all-omitted → `DefaultStopLossATRMult`=1.0. Co-located `OpenStrategy`+`CloseStrategy` (**single** `*StrategyRef` since #842; `UnmarshalJSON` lifts a legacy length-1 `close_strategies` array, rejects len>1; `closeRefs()` feeds 0/1-element scans; `close_strategy` canonical, `close_strategies` still read); **new close evaluator → `closeStrategyOwnedKeys`**. Unknown-key guard. `strategyUsesTieredTPATRClose(sc)` gates on-chain TPs — NOT `len(strategyTPTiers)>0`. `validatePostTPStopLossRulesWithLabels`: `sl_after` fixed-SL gate accepts `stop_loss_atr_mult`, `stop_loss_atr_regime` (non-zero), `stop_loss_pct`, or `stop_loss_margin_pct`.
- `close_defaults.go` — **#866 `user_close_defaults`**: three-layer resolution (system→user→strategy); `applyUserCloseDefaults` after per-strategy normalization (explicit `tp_tiers` wins). `closeDefaultsSupported` = `tiered_tp_pct`,`tiered_tp_atr`,`_live`,`trailing_tp_ratchet`,`_regime` variants; group fns `defaultRegimeATRTiersForGroup`/`defaultTrailingRatchetTiersForRegime`. `validateUserCloseDefaults`: known-name + `tp_tiers` + no stray keys. No version bump. Backtest `--defaults system|user`.
- `state.go`/`db.go` — SQLite-only (`modernc.org/sqlite`); idempotent migrations; tables incl. `trades`,`positions`,`option_positions`,`kill_switch_events`,`pending_manual_actions`,`pending_limit_orders` (#883). `CheckStatePresence` (`GO_TRADER_ALLOW_MISSING_STATE=1`). `ValidatePerpsDirectionConfig` startup check.
- `risk.go`/`strategy_interval.go` — `CheckRisk(*PlatformRiskAssist)` skips `manual`; `effectiveStrategyIntervalSeconds` accelerates checks in DD warn band (DD > `warn_threshold_pct`). **#1008** `forceCloseAllPositions` labels close legs via `classifyPositionTradeType` (HL/OKX perps + HL `manual` with `Multiplier=1` → `perps`; TopStep/CME → `futures`; `Multiplier=0` → `spot`) — operator-display only (`tradeLedgerDeltaSQL` ignores `trade_type`). **#1009** `closePositionIsCorrupt` (qty≤0 OR avgCost≤0) → `forceCloseAllPositions`/`bookPerpsCloseWithFillFee` (portfolio.go) clear with a **zero-PnL** `*_corrupt` leg (cash untouched) so booked PnL reconciles with the closed_positions row.
- `portfolio_warning.go` — **#904 enriched portfolio warning DMs**: `BuildPortfolioWarningMessage(PortfolioWarningMessageInputs)` → triage block (top-N contributors, trend `STABLE`/`WORSENING`/`RECOVERING`, distance to kill switch, recent activity, recommendation). `portfolioWarningMaxRows=5`, `portfolioWarningMaxChars=1900`.
- `circuit_breaker_alert.go` — **#905 enriched CB DMs**: `snapshotPerStrategyCircuitBreaker` (closed/open positions + pending closes) → `formatPerStrategyCircuitBreakerBlock(perStrategyCircuitBreakerFormatInput)` rich alert (trigger, label, portfolio impact, perps context, position/trade tables, recommendation). `circuitBreakerAlertMaxRows=5`, `circuitBreakerAlertMaxChars=1900`.
- `hyperliquid_fills.go` — fill lookup `buildCachedHyperliquidReconcileFillResolver` (built **outside `mu.Lock`**; failure → modeled fee); `HLFillLookup.Px` is VWAP.
- `backfill_hl_fees.go` — `go-trader backfill hl-fees [--strategy <id>|--all] [--apply] [--reset-cash]`; rewrites `exchange_fee=0` rows, replays `strategies.cash`; dry-run default; refuses `--apply` when another alive.
- `regime.go`/`regime_multi_window.go` — `regimeBlocksOpen` blocks entries when `current ∉ allowed && posQty==0`; `stampPositionRegimeIfOpened` runs on **all 5 execute dispatches** (incl. manual; post-TP SL adjustment deferred until after stamp; strategy-level regime synced for display); **`RegimePayload.Label` falls back to `"default"` with no explicit windows** (fixes `regime_directional_policy`/`allowed_regimes`).
- `regime_store.go`/`ui_regime.go` — **#879 global per-cycle regime store**: `regimeBundleKey{Platform,Symbol,Timeframe,SpecJSON}` dedupes; `startRegimeStorePopulation` before check fan-out (parallel, `pythonSemaphore`); **failure policy**: failed bundle → empty `RegimePayload` → gate fails open, `regime=-`, NO inline fallback; sustained → `scriptFailureTracker`. `PayloadForStrategy`/`InjectionJSONForStrategy` read. `check_regime.py` dedicated subprocess, probed at startup. `/api/regime`: window snapshots + `adx3`/`composite7` views.
- `regime_divergence.go` — **#907 `regime_window_divergence`**: HL perps live only. `RegimeWindowDivergence{ShortWindow,MediumWindow,OnDivergence}`; `classifyRegimeDivergence` → hard (bullish+bearish) / soft / none; `applyRegimeDivergenceOverride` mutates local `sc.Direction` after `applyRegimeDirectionalPolicy` (flat=current; open=hold-on-transition); modes `trust_short`/`trust_medium`/`alert_only`; `ranging_directional` from `return_eff`. `RegimeDivergenceState.CyclesActive` in-memory; `/status`+DMs+dashboard badge. SIGHUP-blocked while open.
- `regime_atr.go`/`post_tp_sl.go` — `RegimeATRBlock` backs `*_atr_regime` + `sl_after` ATR variants; `TPATRFraction`/`TPATRFractionRegime *RegimeFloatBlock` resolve via `TrendRegime`. `isTieredTPATRCloseName` gates all five close names (`tiered_tp_atr`…`tiered_tp_atr_live_regime_dynamic`). **Critical:** `parseRegimeTPTiers` must strip `sl_after` before ATR-block parsing — else load failure AND silent never-arm. New `sl_after` surface → `closeStrategyOwnedKeys`. **v15:** `tp_tiers` runtime-only (legacy `tiers` on-disk); `atr_multiple` canonical (legacy `atr`/`multiple`/`fraction`→`close_fraction` read; both-set rejected); `optimizer.py` emits legacy (canonicalized at load).
- `regime_dynamic_tp_sl.go` — **#843 `tiered_tp_atr_live_regime_dynamic`**: live ATR-regime re-resolution on HL perps (`strategyCurrentATRRegime`, hysteresis via `regime_confirm_cycles`, `Position.RegimeAppliedLabel`/`RegimePending*`, cancel+replace via `--force-sl-replace`/`--force-tp-replace-json`); paper uses Python evaluator; backtester rejects.
- `regime_unified.go` — **#841 unified per-regime close block**: `trend_regime` maps each label → `{stop_loss_atr?, tp_tiers:[…]}` (scalar-only sl_after). `closeParamsAreUnifiedRegime` detector; `validateUnifiedRegimeClose` (exhaustive labels). Block **owns the SL** via `unifiedCloseStopLossATR`; `validateUnifiedCloseSoleOwner` rejects strategy-level stop fields; `EffectiveStopLossPct` defers. Hot-reload gated as unit (`unifiedCloseParamsEqualForReload`). Python mirror `regime_atr.py`.
- `trailing_tp_ratchet.go` — trailing-ATR-stop close; each cleared tier tightens `pos.PostTPTrailingATRMult` monotonically + optionally closes `close_fraction` (cumulative). **NO on-chain TP** — EXCLUDED from `isTieredTPATRCloseName`/`strategyUsesTieredTPATRClose`/suppression. Per-tier trail: `trailing_mult_after` (absolute) XOR `tp_atr_fraction` (×firing-tier mult), `resolveTrailingMultAfter`; monotonic at load. Clearing `applyTrailingTPRatchet`/`applyTrailingTPRatchetToPosition`; `SLAdjustedTiersProcessed` watermark. **Scalar form:** SL owner `trailing_stop_atr_mult` (≥ first rung, ≥1.5; rejects fixed-SL + `trailing_stop_pct`/`trailing_stop_atr_regime`). **Regime form:** `tp_tiers` keyed `{label:[tiers]}`; SL owner `trailing_stop_atr_regime`; frozen at open. **Scope: HL perps + `manual`.** Hot-reload blocked while open; owned keys `tp_tiers`,`use_defaults`. Paper+backtest via `trailing_tp_ratchet.py`. **Defaults** (`use_defaults`/omit): `1.5×/1.5×/0.8×` at `2×/2.5×/3×` ATR (regime: per-group); `DEFAULT_RATCHET_TIERS`.
- `regime_directional_policy.go` — `applyRegimeDirectionalPolicy` mutates **local** `sc` (flat=`result.Regime`, open=`pos.Regime`); all 3 canonical labels required; **HL perps only, `regime.enabled=true` required**. **Backtester parity (#1025):** `run_backtest.py --config` replays the same per-cycle direction/invert resolver, using the shifted prior-bar regime while flat and the stamped position regime while open. On the open/close engine path the open-action gate is resolved **after** the close leg (#1025 review): a same-bar full close→reopen re-resolves the new entry from the now-flat book's current-bar regime, not the just-closed position's frozen regime (a surviving partial-close leg keeps `position != 0`, so its freeze is preserved and the open block can't fire). **Orphan auto-close (#822):** `perpsRegimeDirectionOrphanConflict` + `runRegimeDirectionOrphanCloses` (outside `mu`, 90s budget) detect sole-owner positions conflicting effective direction (incl. static-direction orphans; `regime.enabled` not required; `direction="both"` never conflicts). Trades `regime_direction_flip`. Lags one cycle.
- `regime_profile_allocation.go` — **#998 M4 regime-profile allocation** (opt-in; HL perps live+paper, `regime.enabled` required): a slow long-window regime label picks one of two validated open-param profiles, switched hysteretically (`confirm_bars`, WARN<12) and **flat-only** (frozen to `Position.OpenProfile` while open). Config `regime_profile_allocation{window, profiles(label→name, all classifier labels), param_sets(exactly 2), confirm_bars≥1, initial_profile}` (no version bump). `resolveRegimeProfile` (pure closed-bar state machine) → `applyRegimeProfileParams` merges `param_sets[active]` over loop-local `sc.OpenStrategy.Params` (never mutates cfg) before check; `stampPositionProfileIfOpened` freezes on open; `RegimeProfileState`. Backtestable: `run_backtest.py --config` / `eval_windows.py --profile-allocation` (`_ProfileSwitcher`, label series shifted 1 bar).
- `scale_in.go` — **#873 scale-in / pyramiding** (opt-in; HL perps + manual, live + paper). Same-direction signal on an open position ADDS size. `applyScaleIn(pos,addQty,addPrice)` blends `AvgCost`/`Quantity`, grows `InitialQuantity`, **FREEZES** `EntryATR`/`Regime` + cleared-tier watermark; sets `ScaleInResizePending`. **Frozen SL/TP geometry** via `Position.RiskAnchorPrice`/`pos.riskAnchorPrice()` — all ATR-geometry/trigger/display sites read anchor, NOT blended AvgCost. `perpsScaleInDecision` gates `allow_scale_in` + `scale_in.{max_adds,max_added_notional_usd,add_spacing_atr}` (signed; >0 winners, <0 avg-down). **Trailing-SL** `scaleInResizeTrailingSLNow` grows SL same cycle; `ScaleInResizePending` cleared by non-trailing owners only. Add leg `TradeType:"scale_in"`, excluded from `#T`. State cols `scale_in_count`/`last_add_price`/`added_notional_usd`/`risk_anchor_price`/`scale_in_resize_pending`. Hot-reload when flat. **NOT backtested.**

## Key Pattern mechanics (detail)

These expand the guardrail one-liners in CLAUDE.md § Key Patterns.

- **Bidirectional perps roster** — short-entry strategies that must be in `bidirectionalPerpsStrategies` (init.go): `triple_ema_bidir`,`bear_pullback_st`,`vwap_rejection_st`,`donchian_breakout`,`chart_pattern`,`liquidity_sweeps`,`momentum_pro`,`mean_reversion_pro`,`consolidation_range`,`mtf_confluence`,`vol_momentum`,`funding_skew`,`regime_adaptive`. Flip sizing `perpsLiveOrderSize`; catastrophic flip → close-only. (`regime_adaptive_htf` ships long/flat — NOT in the list.) **#1009:** a flip requires `closeFraction==0` (any close action is close-only — `perpsLiveOrderSize` + its `runHyperliquidExecuteOrder` mirror must agree or `net_new_sz` diverges); `perpsCloseActionSuppressesNewSL` (full close `closeFraction==1.0` OR directional-gate close) gates arming a new reduce-only SL; `executePerpsSignalWithLeverage` caps `closeQty` at `pos.Quantity` (no negative residual).

- **Regime-model promotion gate — bounded-window ADX re-validation (#1082) `regime_bounded_window_validate.py`** — gates promoting a fitted regime model into the live classifier (#1074) on a *live-faithful* re-score. The offline fit (`regime_calibrate.fit_on_window`→`composite_feature_matrix`→`compute_regime`) seeds recursive Wilder ADX at the **full eval-window start**; live (`shared_scripts/check_regime.py`, `--ohlcv-limit` default 200, a fresh subprocess per cycle) re-runs over a **bounded** fetch, so the same bar's ADX can differ live vs in the fit. All cases surface a `provenance` report block + loud stderr line, so an in-sample re-score is never *silently* promoted. Only the `adx` feature is path-dependent (`standard_atr` is a simple rolling mean, not Wilder), but the harness still recomputes the whole feature vector per window so it stays faithful if that changes.
