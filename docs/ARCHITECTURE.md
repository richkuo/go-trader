## `scheduler/` file-ownership map

`scheduler/` is one Go `package main`.

## Key Pattern mechanics (detail)

These expand the guardrail one-liners in CLAUDE.md § Key Patterns.

- **Bidirectional perps roster** — short-entry strategies that must be in `bidirectionalPerpsStrategies` (init.go): `triple_ema_bidir`,`bear_pullback_st`,`vwap_rejection_st`,`donchian_breakout`,`chart_pattern`,`liquidity_sweeps`,`momentum_pro`,`mean_reversion_pro`,`consolidation_range`,`mtf_confluence`,`vol_momentum`,`funding_skew`,`regime_adaptive`. Flip sizing `perpsLiveOrderSize`; catastrophic flip → close-only. (`regime_adaptive_htf` ships long/flat — NOT in the list.) **#1009:** a flip requires `closeFraction==0` (any close action is close-only — `perpsLiveOrderSize` + its `runHyperliquidExecuteOrder` mirror must agree or `net_new_sz` diverges); `perpsCloseActionSuppressesNewSL` (full close `closeFraction==1.0` OR directional-gate close) gates arming a new reduce-only SL; `executePerpsSignalWithLeverage` caps `closeQty` at `pos.Quantity` (no negative residual).
- **Regime detection** (`shared_tools/regime.py`): `adx` (3 labels) / `composite` (7). `validateStrategyRegimeVocabulary` matches keys to each window's classifier. SIGHUP: classifier swap on referenced window blocked while open. **#861** `efficiency` threshold in Python `_DEFAULT_COMPOSITE_THRESHOLDS` AND Go `RegimeCompositeThresholds` — keep in sync. **#1062 `regime.display_windows`:** display-only summary filter; names must exist in `regime.windows` (typo rejects); hot-reloadable via `regimeConfigEqualIgnoringDisplayWindows`; never affects gating.
- **HL trailing stop-loss** — cancel+replace when trigger moves > `TrailingStopMinMovePct` (0.5%); `hlSLEffectiveQty=min(virtualQty,onChainQty)`. **#1121** `RatchetFallbackNormalizePending` one-shot widen. Protection snapshot must carry regime + #873 anchor (#1015).
- **HL margin** — `MarginMode` (`isolated`|`cross`, default `isolated`); `update_leverage` from flat only (HL rejects on open). Multi-strategy same coin: `hyperliquidPeerStrategyErrors` requires shared `margin_mode`+`leverage`; multiple trailing-stop owners (incl. `manual`) allowed — live replacement is serialized per coin and shared-coin reconciliation books only exact OID-confirmed SL fills. Sub-account = full independence.
- **HL on-chain protection suppression** — `strategyConfigWithOnChainProtectionFilter(sc)` nils the single `CloseStrategy` when `closeStrategySuppressedByOnChainProtection(sc)` (close is `tiered_tp_atr`/`tiered_tp_atr_live` on HL **live** perps placing on-chain TPs) — prevents limit-fill/close-evaluator race. **Paper never suppressed.** New ATR-tiered evaluator → add to `closeStrategiesSuppressedByOnChainProtection`.
- **HL perps stop-loss priority** — `EffectiveStopLossPct(sc)`: ATR/regime → `0` at open → trailing/scalar pct → margin/leverage → `min(MaxDrawdownPct,50)` (seven mutually-exclusive owners, config.go). Fixed ATR `AvgCost ± mult*EntryATR`; regime-aware from `pos.Regime`. Hot-reload: numeric change OK on fresh open; nil↔positive AND scalar↔regime blocked while open. HL caps 1000 triggers/acct.
- **SIGHUP hot reload** (`applyHotReloadConfig`): `validateHotReloadCompatible` blocks strategy add/remove, script/args/type/platform/HTFFilter, kill-switch identity, DB path. `validateHotReloadStateCompatible` blocks **while open** `leverage`/`direction`/`invert_signal`/`sl_after`/`*_atr_regime`/`regime_directional_policy`/`regime_window_divergence`/unified-per-regime-close block. When flat, `regime_directional_policy`/`regime_window_divergence` apply immediately. `invert_signal` (HL perps/manual) flips BUY↔SELL via `applySignalInversion`, before `Direction`.

## Backtest harnesses (detail)

Expands CLAUDE.md § Backtest. The run commands and look-ahead/`--config` guardrails stay there; the mechanics live here.

- **Close registry kwargs:** `--config` reads single `close_strategy`; **#951** gates `config_version>=15`. Entry ATR guard 50%-of-AvgCost; bar-level only.
- **Regime / HL-live-only:** entries gated by `allowed_regimes`; **#1025** threads from `--config`; **#1058** composite via `regime.windows`; **#1085** directional policy behind cert gate. Rejects at init: `regime_window_divergence`, `tiered_tp_atr_live_regime_dynamic`. `regime_directional_policy` backtestable via #1025 resolver + #1085 gate.
- **#951 entry-transform:** `direction`+`invert_signal` in `Backtester.run`; path-aware (engine vs plain). Rejects `both` without close evaluator; stray `invert_signal` on non-perps/manual.
- **#989 short-leg:** `eval_windows.py --direction short` on plain single-leg path.
- **Funding (#960/#988):** per-bar `funding_rate` via `funding_fetcher` (`merge_asof` backward); #988 `funding_accrual` carry for `delta_neutral_funding`.
- **#1005 liquidation floor:** sticky-floors equity at 0; `±LIQUIDATED_METRIC_FLOOR=100.0` on blown metrics; propagated by `eval_windows`/`fee_audit`.
- **M1–M6 research harnesses** (import `eval_windows.py` `WINDOWS`/`DATASETS`/`PLATFORM`):
  - **M1** `eval_windows.py` — incumbent-relative multi-window validator.
  - **M2** `run_backtest.py --mode optimize --sweep-close` — joint open×exit walk-forward.
  - **M3** `exit_diagnostics.py` — MFE/MAE/bleed-mode; default-off `close/{time_stop,atr_stop,zscore_target}.py` (live deferred).
  - **M5** `fee_audit.py` — net-vs-gross fee drag; `salvage_verdict`; short-capable strategies need `--direction short`.
